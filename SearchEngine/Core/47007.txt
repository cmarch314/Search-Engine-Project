\t\t\t\tAbstraction Matching    \t\t\t</h1><!-- .entry-title -->\r\n  \t\t\t</header>\r\n\r\n  \t\t\t\r\n  \t\t\t\r\n  \t\t\t<div class=\"entry-content clearfix\">\r\n    \t\t\t<p>In a discussion yesterday, a thought occurred:  many problems in different areas are about selecting proper abstractions to match applications.  Abstraction matching is a challenging and interesting problem.</p>\n<p>In computer science particularly, abstraction matching is very important.  I hear long discussion whether something that can be done with a new abstraction, say &#8216;events&#8217; can be done with an oldabstraction say &#8216;objects&#8217;.  This discussion can soon degenerate if somebody starts arguing whether that thing can be done by Turing Machine or not.  If something is computable than what abstractions you use are more important if you consider efficiency and human easeof relevant application oriented operations.  A particularly interesting case of this is the Relational Database Model.  This abstraction is very unnatural but once you consider the operations in data collections of &#8216;structured data&#8217;, it turns out to be a very good representation.  That is because the abstraction matches the application.</p>\n<p>In many cases one can only evaluate effectiveness of an abstraction after it has been implemented and tested in applications.  It is very rare that one can mathematically evaluate this in emerging complex applications.</p>\n<p>I need to think more about this, but so far it appears that Abstraction Matching may indeed, like impedance matching, an important issue.</p>\n  \t\t\t</div>\r\n  \t\t\t<div class=\"entry-meta-bar clearfix\">\t        \t\t\t\r\n    \t\t\t<div class=\"entry-meta\">\r\n    \t\t\t\t<span class=\"by-author\">By <a href=\"http://ngs.ics.uci.edu/author/ramesh/\">Ramesh</a></span> |\r\n    \t\t\t\t<span class=\"date\"><a href=\"http://ngs.ics.uci.edu/abstraction-matching/\" title=\"8:06 am\">September 11, 2008</a></span> |\r\n    \t\t\t\t             \t\t<span class=\"category\"><a href=\"http://ngs.ics.uci.edu/category/technical-thoughts/\" rel=\"category tag\">Technical Thoughts</a></span> |\r\n             \t \r\n    \t\t\t\t             \t\t<span class=\"comments\"><a href=\"http://ngs.ics.uci.edu/abstraction-matching/#comments\" title=\"Comment on Abstraction Matching\">1 Comment</a></span> |\r\n             \t \t\t          \t\t\t\t\r\n    \t\t\t</div><!-- .entry-meta -->\r\n    \t\t</div>\r\n\r\n  \t\t\t\t\t\t<ul class=\"default-wp-page clearfix\">\r\n\t\t\t\t<li class=\"previous\"><a href=\"http://ngs.ics.uci.edu/interactive-digital-media-in-singapore/\" rel=\"prev\"><span class=\"meta-nav\">&larr;</span> Interactive Digital Media in Singapore</a></li>\r\n\t\t\t\t<li class=\"next\"><a href=\"http://ngs.ics.uci.edu/going-back-to-usa/\" rel=\"next\">Going Back to USA &#8212;
