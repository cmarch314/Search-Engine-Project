class=\"level1\">\n\n<p>\n\nThis is more of a random list of how-to topics than a tutorial.\n</p>\n\n</div>\n\n<h2><a name=\"basics\" id=\"basics\">Basics</a></h2>\n<div class=\"level2\">\n\n<p>\n\nThere are three main categories of file for Ipe: \n\n</p>\n<ul>\n<li class=\"level1\"><div class=\"li\"> Ipe document (.ipe, .pdf, .ps, etc.)</div>\n</li>\n<li class=\"level1\"><div class=\"li\"> Style sheets (.isy)</div>\n</li>\n<li class=\"level1\"><div class=\"li\"> Ipelets (.lua)</div>\n</li>\n</ul>\n\n</div>\n\n<h3><a name=\"ipe_document\" id=\"ipe_document\">Ipe Document</a></h3>\n<div class=\"level3\">\n\n<p>\n\nIpe can save and open documents in a variety of formats, including xml, pdf, and postscript. See the <a href=\"http://ipe7.sourceforge.net/manual/manual_41.html\" class=\"urlextern\" title=\"http://ipe7.sourceforge.net/manual/manual_41.html\"  rel=\"nofollow\">Ipe Manual</a> for more details.\n</p>\n\n<p>\nYou can use <code>ipetoipe</code> to convert between these different formats. Sometimes, this might be necessary to fix problems with the program:\n</p>\n\n</div>\n\n<h4><a name=\"quote_character_in_a_layer_name\" id=\"quote_character_in_a_layer_name\">Quote character (&quot;) in a layer name</a></h4>\n<div class=\"level4\">\n\n<p>\n\nNever put quote characters (\u201d) in the name of a layer. Ipe will crash when you try to open such a document.\n</p>\n\n</div>\n\n<h5><a name=\"how_to_fix_it\" id=\"how_to_fix_it\">How to fix it:</a></h5>\n<div class=\"level5\">\n\n<p>\n\nInternally, Ipe saves files as xml. Even if you save it as a pdf, Ipe will embed the xml inside the pdf. Ipe must be able to parse this xml in order to properly load a file. However, if you put (\u201d) characters in places that get written as metadata, Ipe will not be able to parse the xml. You will get poorly formatted xml that looks like this:\n</p>\n<pre class=\"code\">\n&lt;page title=&quot;&amp;apos;this&amp;quot; is fixed for titles&quot;&gt;\n&lt;layer name=&quot;don&#039;t_do_&quot;this&quot;&quot;/&gt;\n&lt;view layers=&quot;don&#039;t_do_&quot;this&quot;&quot; active=&quot;don&#039;t_do_&quot;this&quot;&quot;/&gt;\n&lt;text layer=&quot;don&#039;t_do_&quot;this&quot;&quot; transformations=&quot;translations&quot; pos=&quot;128 704&quot; stroke=&quot;black&quot; type=&quot;label&quot; width=&quot;36.862&quot; height=&quot;6.128&quot; depth=&quot;0&quot; valign=&quot;baseline&quot;&gt;test text&lt;/text&gt;\n&lt;/page&gt;\n</pre>\n\n<p>\n\nWhen you try and open this file, you will get an error like this:\n\n</p>\n<pre class=\"code\">  Document &#039;/home/jsimons/Desktop/bad_ipe.ipe&#039; could not be opened\n  Parsing error at position 6051</pre>\n\n<p>\n\nIpe is helpfully trying to tell you which character the xml parser failed on. To fix it, you have to manually edit the xml. If you saved as .ipe, then just open up the file in your favorite text editor and remove the extra quotes from the xml. If you saved it as a pdf, you first have to extract the xml from the pdf (since in the pdf it&#039;s saved as a byte stream, you can&#039;t see the plain text). One easy way to do this is to run\n\n</p>\n<pre class=\"code\">ipetoipe -xml input.pdf output.ipe</pre>\n\n</div>\n\n<h3><a name=\"style_sheets\" id=\"style_sheets\">Style sheets</a></h3>\n<div class=\"level3\">\n\n<p>\n\nStylesheets are text files that can be included to customize various formatting aspects of an Ipe document. For example, you can use it to define more colors, add transparency levels, or create gradients. Stylesheets work a bit like preambles in LaTeX. Multiple stylesheets can be used in one Ipe document, and a single stylesheet can be reused over multiple documents.\n</p>\n\n<p>\nBy default, Ipe uses a basic style sheet <code>basic.isy</code>. To use your own stylesheet, you can either have Ipe automatically load it for every document, or you add it manually for each document. You would want to use the former option for things that are useful across documents, like color definition and transparency levels, and the latter option for things specific to the document, like backgrounds and location of the title text.\n</p>\n\n<p>\nRegardless of how you have Ipe load the stylesheet, if the stylesheet changes, you must do <code>Edit &gt; Update style sheets</code> for Ipe to see the changes.\n</p>\n\n</div>\n\n<h4><a name=\"loading_stylesheets_automatically\" id=\"loading_stylesheets_automatically\">Loading stylesheets automatically</a></h4>\n<div class=\"level4\">\n\n<p>\n\nHere&#039;s how we can add and change the default stylesheets loaded by Ipe.\n\n</p>\n<ul>\n<li class=\"level1\"><div class=\"li\"> Add the stylesheets you want into a styles directory. Typically this is <code>~/.ipe/styles</code> (you may need to create the directory yourself).</div>\n</li>\n<li class=\"level1\"><div class=\"li\"> In your customization Ipelet, add the names of all the stylesheets you want to load automatically: <pre class=\"code\">\nprefs.styles = { &quot;basic&quot;, &quot;good_colors&quot; }\n</pre>\n</div>\n</li>\n</ul>\n\n</div>\n\n<h4><a name=\"adding_stylesheets_manually\" id=\"adding_stylesheets_manually\">Adding stylesheets manually</a></h4>\n<div class=\"level4\">\n\n<p>\n\nThe workflow of using a stylesheet specific to a document:\n\n</p>\n<ol>\n<li class=\"level1\"><div class=\"li\"> Create or open a stylesheet in a text editor. Here is a barebones example: <dl class=\"file\">\n<dt><a href=\"/_export/code/wiki:ipe_tutorial_introduction?codeblock=3\" title=\"Download Snippet\" class=\"mediafile mf_isy\">my-stylesheet.isy</a></dt>\n<dd><pre class=\"code file xml\"><span class=\"sc3\"><span class=\"re1\">&lt;ipestyle</span> <span class=\"re0\">name</span>=<span class=\"st0\">&quot;my-stylesheet&quot;</span><span class=\"re2\">&gt;</span></span>\n<span class=\"sc-1\">&lt;!--ipe customization--&gt;</span>\n<span class=\"sc3\"><span class=\"re1\">&lt;/ipestyle<span class=\"re2\">&gt;</span></span></span></pre>\n</dd></dl>\n</div>\n</li>\n<li class=\"level1\"><div class=\"li\"> Edit it. Some examples of things to add can be found in this tutorial.</div>\n</li>\n<li class=\"level1\"><div class=\"li\"> Save it.</div>\n</li>\n<li class=\"level1\"><div class=\"li\"> Include it in the Ipe document: in Ipe, go to <code>Edit &gt; Style sheets &gt; Add</code>.</div>\n</li>\n<li class=\"level1\"><div class=\"li\"> Update it: in Ipe, click <code>Edit &gt; Update style sheets</code>. This must be done every time an included stylesheet is modified.</div>\n</li>\n</ol>\n\n</div>\n\n<h2><a name=\"customizations\" id=\"customizations\">Customizations</a></h2>\n<div class=\"level2\">\n\n<p>\n\nYou can use Ipelets to customize the behavior of Ipe.\n<a href=\"http://ipe7.sourceforge.net/manual/manual_40.html\" class=\"urlextern\" title=\"http://ipe7.sourceforge.net/manual/manual_40.html\"  rel=\"nofollow\">http://ipe7.sourceforge.net/manual/manual_40.html</a>\n</p>\n\n<p>\nThe customization script is written as an Ipelet. The easiest way to do the customization is to copy <code>prefs.lua</code>, <code>shortcuts.lua</code>, and <code>mouse.lua</code> in the Ipe lua directory to your Ipelets directory and modify them to your liking. (Use <code>-show-configuration</code> to see where these directories are.)\n</p>\n\n<p>\nThe <code>mouse.lua</code> is also good for discovering all the useful mouse shortcuts in Ipe.\n</p>\n\n</div>\n\n<h2><a name=\"example_stylesheets\" id=\"example_stylesheets\">Example Stylesheets</a></h2>\n<div class=\"level2\">\n\n</div>\n\n<h3><a name=\"color-blind_colors\" is an example code snippet to include in your stylesheet:\n</p>\n<pre class=\"code xml\"><span class=\"sc3\"><span class=\"re1\">&lt;gradient</span> <span class=\"re0\">name</span>=<span class=\"st0\">&quot;mygradient&quot;</span> <span class=\"re0\">type</span>=<span class=\"st0\">&quot;axial&quot;</span> <span class=\"re0\">extend</span>=<span class=\"st0\">&quot;yes&quot;</span> <span class=\"re0\">coords</span>=<span class=\"st0\">&quot;0 0 0 600&quot;</span><span class=\"re2\">&gt;</span></span>\n  <span class=\"sc3\"><span class=\"re1\">&lt;stop</span> <span <span class=\"sc3\"><span class=\"re1\">&lt;stop</span> <span class=\"re0\">offset</span>=<span class=\"st0\">&quot;0.48&quot;</span> <span class=\"re0\">color</span>=<span class=\"st0\">&quot;1 1 class=\"re2\">/&gt;</span></span>\n<span class=\"sc3\"><span class=\"re1\">&lt;/gradient<span class=\"re2\">&gt;</span></span></span></pre>\n\n<p>\nAxial means the gradient is along a single direction. The other option is radial (needs to checked). The coordinates specify the two endpoints of the direction vector. The stops represent the different points along the direction vector at which color should change.\n</p>\n\n<p>\nOnce the stylesheet uploaded, the gradient option should show up in the context menu of objects that can have a gradient fill.\n</p>\n\n</div>\n\n<h2><a name=\"ipelets\" id=\"ipelets\">Ipelets</a></h2>\n<div class=\"level2\">\n\n<p>\n\nYou can expand the existing functionality of Ipe by using Ipelets, which are <a href=\"http://www.lua.org/\" class=\"urlextern\" title=\"http://www.lua.org/\"  rel=\"nofollow\">Lua</a> scripts that Ipe runs on-the-fly. The installed Ipelets can be accessed and used under the \u201cIpelets\u201d menu item.\n</p>\n\n<p>\nEach Ipelet consists of a single Lua source file with other support files. To install an Ipelet, simply save the files in your Ipelets directory. You can run <code>ipe -show-configuration</code> to find out where that directory is (typically, this is <code>~/.ipe/ipelets</code>).\n</p>\n\n<p>\nThe bottom of the <a href=\"http://lamut.informatik.uni-wuerzburg.de/mediawiki/ipe7/index.php/Main_Page#Ipelets\" class=\"urlextern\" title=\"http://lamut.informatik.uni-wuerzburg.de/mediawiki/ipe7/index.php/Main_Page#Ipelets\"  rel=\"nofollow\">Ipe Wiki</a> has a good list of Ipelets. They include things like automatic page numbering, background boxes for text, and graph drawing tools (where edges will follow when vertices are dragged).\n</p>\n\n</div>\n\n<h3><a name=\"cgal_ipelets\" id=\"cgal_ipelets\">CGAL Ipelets</a></h3>\n<div class=\"level3\">\n\n<p>\nThis is assuming that Ipe is already installed and runs successfully.\n</p>\n\n<p>\nOn debian:\n</p>\n<pre class=\"code\">0. Make sure you have cmake\n    * $ sudo apt-get install cmake \n1. Get the required cgal library packages and all their dependencies\n    * $ sudo apt-get install libcgal-dev\n2. Get the demos and examples files\n    * $ sudo apt-get install libcgal-demo\n    * For me, the demo was installed to /usr/share/doc/libcgal-demo/\n3. Unpack the demo tar somewhere reasonable\n    * $ mkdir ~/CGAL\n    * $ cd ~/CGAL\n    * $ tar xzf /usr/share/doc/libcgal-demo/demo.tar.gz\n4. Prepare the demos to be compiled\n    * $ cd demo\n    * $ cmake .\n5. Compile the demo(s). \nTo make just the Ipelets, go to the ready-to-compile subdirectory (yes, it&#039;s another nested level, demo/demo/), and make.\nIt will take a few minutes to compile all the Ipelets\n    * $ cd ~/CGAL/demo/demo/CGAL_ipelets/\n    * $ make\n6. Copy the compiled output &#039;*.so&#039; files and the &#039;*.lua&#039; into the Ipelet directory. \n   On the default 7.1.3 Ipe installation, the command is:\n    * $ sudo find . -name &#039;*.so&#039; -exec cp &#039;{}&#039; /usr/local/lib/ipe/7.1.3/ipelets/ \\;\n    * $ sudo find ~/CGAL/demo/CGAL_ipelets/lua/ -name &#039;*.lua&#039; -exec cp &#039;{}&#039; /usr/local/lib/ipe/7.1.3/ipelets/ \\;</pre>\n\n</div>\n\n<h2><a name=\"presentations\" id=\"presentations\">Presentations</a></h2>\n<div class=\"level2\">\n\n</div>\n\n<h3><a name=\"ipe_presentation_background\" id=\"ipe_presentation_background\">Ipe presentation background</a></h3>\n<div class=\"level3\">\n\n<p>\n\nTo use an Ipe drawing as a background:\n\n</p>\n<ol>\n<li class=\"level1\"><div class=\"li\"> group the objects of the drawing</div>\n</li>\n<li class=\"level1\"><div class=\"li\"> copy the <acronym title=\"Extensible Markup Language\">XML</acronym> code of the group object</div>\n</li>\n<li class=\"level1\"><div class=\"li\"> paste it inside the following code snippet in your Ipe stylesheet:<pre class=\"code xml\"><span class=\"sc3\"><span class=\"re1\">&lt;symbol</span> <span class=\"re0\">name</span>=<span class=\"st0\">&quot;Background&quot;</span> <span class=\"re0\">xform</span>=<span class=\"st0\">&quot;yes&quot;</span><span class=\"re2\">&gt;</span></span>\n<span class=\"sc-1\">&lt;!--insert XML here--&gt;</span>\n<span class=\"sc3\"><span class=\"re1\">&lt;/symbol<span class=\"re2\">&gt;</span></span></span></pre>\n</div>\n</li>\n</ol>\n\n<p>\nThe symbol name has to be <strong>Background</strong> spelled exactly that way. The xform option allows you to only store a single copy of the background in the <acronym title=\"Portable Document Format\">PDF</acronym> file.\n</p>\n\n<p>\nTo turn off the background on a specific page, add a layer called <strong>BACKGROUND</strong>. Setting that layer to visible/invisible seems to have no effect.\n</p>\n\n</div>\n\n<h3><a name=\"ipe_presentation_title\" id=\"ipe_presentation_title\">Ipe presentation title</a></h3>\n<div class=\"level3\">\n\n<p>\n\nAdd something like this to your stylesheet.\n</p>\n<pre class=\"code xml\"><span class=\"sc3\"><span class=\"re1\">&lt;color</span> <span class=\"re0\">name</span>=<span class=\"st0\">&quot;titlecolor&quot;</span> <span class=\"re0\">value</span>=<span class=\"st0\">&quot;1 0.5 0.5&quot;</span><span class=\"re2\">/&gt;</span></span>\n<span class=\"sc3\"><span class=\"re1\">&lt;textsize</span> <span class=\"re0\">name</span>=<span class=\"st0\">&quot;titlesize&quot;</span> <span class=\"re0\">value</span>=<span class=\"st0\">&quot;\\large\\bf&quot;</span> <span class=\"re2\">/&gt;</span></span>\n<span class=\"sc3\"><span class=\"re1\">&lt;textstretch</span> <span class=\"re0\">name</span>=<span class=\"st0\">&quot;titlesize&quot;</span> <span class=\"re0\">value</span>=<span class=\"st0\">&quot;2.8&quot;</span> <span class=\"re2\">/&gt;</span></span>\n<span class=\"sc3\"><span class=\"re1\">&lt;titlestyle</span> <span class=\"re0\">pos</span>=<span class=\"st0\">&quot;25 554&quot;</span> <span class=\"re0\">size</span>=<span class=\"st0\">&quot;titlesize&quot;</span> <span class=\"re0\">color</span>=<span class=\"st0\">&quot;titlecolor&quot;</span> <span class=\"re0\">halign</span>=<span class=\"st0\">&quot;left&quot;</span> <span class=\"re0\">valign</span>=<span class=\"st0\">&quot;baseline&quot;</span> <span class=\"re2\">/&gt;</span></span></pre>\n\n<p>\nThe <code>textsize</code> allows you to specify the LaTeX code for the font you want to use. The <code>textstretch</code> specifies how much to magnify the LaTex generated text, which is useful to enlargen text in a presentation. The <code>titlestyle</code> is where you specify the appearance and position of your title text.\n</p>\n\n<p>\nThe <code>textsize</code> and <code>textstretch</code> can be used alone to define a text style (size and shape). They simply have to have the same <code>name</code> attribute. This is a little hacky because this was only meant to define a text size and not text style.\n</p>\n\n</div>\n\n<h2><a name=\"miscellaneous\" id=\"miscellaneous\">Miscellaneous</a></h2>\n<div class=\"level2\">\n\n</div>\n\n<h3><a name=\"latex_packages\" id=\"latex_packages\">LaTeX packages</a></h3>\n<div class=\"level3\">\n\n<p>\n\nTo include a LaTeX package in Ipe, go to <strong>Edit &gt; Document Properties &gt; Latex preamble</strong> and add the LaTeX command <code>\\usepackage{my-package}</code>.\n</p>\n\n<p>\nAlternatively, add the following in a stylesheet:\n</p>\n<pre class=\"code xml\"><span class=\"sc3\"><span class=\"re1\">&lt;preamble<span class=\"re2\">&gt;</span></span></span>\n\\usepackage{my-package}\n<span class=\"sc3\"><span class=\"re1\">&lt;/preamble<span class=\"re2\">&gt;</span></span></span></pre>\n\n<p>\nYou can also put whatever LaTeX commands usually go in the preamble of a LaTeX document in the <code>&lt;preamble&gt;</code> tags, such as <code>\\input{}</code>, <code>\\newcommand{}</code>, etc.\n</p>\n\n</div>\n\n<h2><a name=\"external_resources\" id=\"external_resources\">External
