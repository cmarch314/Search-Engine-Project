 <div class=\"wikipage searchable\">\n        \n          <div id=\"wikipage\" class=\"trac-content\"><h1 id=\"WikiProcessors\">Wiki Processors</h1>\n<p>\nProcessors are <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/WikiMacros\">WikiMacros</a> designed to provide alternative markup formats for the <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracWiki\">Wiki engine</a>. Processors can be thought of as <em>macro functions to process user-edited text</em>. \n</p>\n<p>\nWiki processors can be used in any Wiki text throughout Trac,\nfor various different purposes, like:\n</p>\n<ul><li><a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/WikiProcessors#CodeHighlightingSupport\">syntax highlighting</a> or for rendering text verbatim,\n</li><li>rendering <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/WikiProcessors#HTMLrelated\">Wiki markup inside a context</a>, \nlike inside &lt;div&gt; blocks or &lt;span&gt; or within &lt;td&gt; or &lt;th&gt; table cells,\n</li><li>using an alternative markup syntax, like <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/WikiHtml\">raw HTML</a> and\n<a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/WikiRestructuredText\">Restructured Text</a>,\nor <a class=\"ext-link\" href=\"http://www.textism.com/tools/textile/\"><span class=\"icon\">\u200b</span>textile</a>\n</li></ul><h2 id=\"UsingProcessors\">Using Processors</h2>\n<p>\nTo use a processor on a block of text, first delimit the lines using\na Wiki <em>code block</em>:\n</p>\n<pre class=\"wiki\">{{{\nThe lines\nthat should be processed...\n}}}\n</pre><p>\nImmediately after the <tt>{{{</tt> or on the line just below, \nadd <tt>#!</tt> followed by the <em>processor name</em>.\n</p>\n<pre class=\"wiki\">{{{\n#!processorname\nThe lines\nthat should be processed...\n}}}\n</pre><p>\nThis is the \"shebang\" notation, familiar to most UNIX users.\n</p>\n<p>\nBesides their content, some Wiki processors can also accept <em>parameters</em>,\nwhich are then given as <tt>key=value</tt> pairs after the processor name, \non the same line. If <tt>value</tt> has to contain space, as it's often the case for\nthe style parameter, a quoted string can be used (<tt>key=\"value with space\"</tt>).\n</p>\n<p>\nAs some processors are meant to process Wiki markup, it's quite possible to\n<em>nest</em> processor blocks.\nYou may want to indent the content of nested blocks for increased clarity,\nthis extra indentation will be ignored when processing the content.\n</p>\n<h2 id=\"Examples\">Examples</h2>\n<table class=\"wiki\">\n<tr><th> Wiki Markup </th><th> Display \n</th></tr><tr><td colspan=\"2\" align=\"center\" style=\"border: none\"><blockquote>\n<p>\n<span class=\"underline\">Example 1</span>: Inserting raw HTML\n</p>\n</blockquote>\n</td></tr><tr>\n<td style=\"border: none\"><pre class=\"wiki\">{{{\n&lt;h1 style=\"color: grey\"&gt;This is raw HTML&lt;/h1&gt;\n}}}\n</pre></td><td style=\"border: none; padding-left: 2em\" valign=\"top\"><h1 style=\"color: grey\">This is raw HTML</h1>\n</td></tr><tr>\n<td colspan=\"2\" align=\"center\" style=\"border: none\"><blockquote>\n<p>\n<span class=\"underline\">Example 2</span>: Highlighted Python code in a &lt;div&gt; block with custom style\n</p>\n</blockquote>\n</td></tr><tr>\n<td style=\"border: none\"><pre class=\"wiki\">{{{#!div style=\"background: #ffd; border: 3px ridge\"\n\nThis is an example of embedded \"code\" block:\n\n  {{{\n  #!python\n  def hello():\n      return \"world\"\n  }}}\n\n}}}\n</pre></td><td style=\"border: none; padding: 1em\" valign=\"top\"><div style=\"background: #ffd; border: 3px ridge\" class=\"wikipage\"><p>\nThis is an example of embedded \"code\" block:\n</p>\n<div class=\"code\"><pre><span class=\"k\">def</span> <span class=\"nf\">hello</span><span class=\"p\">():</span>\n    <span class=\"k\">return</span> <span class=\"s\">\"world\"</span>\n</pre></div></div></td></tr><tr>\n<td colspan=\"2\" align=\"center\" style=\"border: none\"><blockquote>\n<p>\n<span class=\"underline\">Example 3</span>: Searching tickets from a wiki page, by keywords.\n</p>\n</blockquote>\n</td></tr><tr>\n<td style=\"border: none\"><pre class=\"wiki\">{{{\n#!html\n&lt;form action=\"/query\" method=\"get\"&gt;&lt;div&gt;\n&lt;input type=\"text\" name=\"keywords\" value=\"~\" size=\"30\"/&gt;\n&lt;input type=\"submit\" value=\"Search by Keywords\"/&gt;\n&lt;!-- To control what fields show up use hidden fields\n&lt;input type=\"hidden\" name=\"col\" value=\"id\"/&gt;\n&lt;input type=\"hidden\" name=\"col\" value=\"summary\"/&gt;\n&lt;input type=\"hidden\" name=\"col\" value=\"status\"/&gt;\n&lt;input type=\"hidden\" name=\"col\" value=\"milestone\"/&gt;\n&lt;input type=\"hidden\" name=\"col\" value=\"version\"/&gt;\n&lt;input type=\"hidden\" name=\"col\" value=\"owner\"/&gt;\n&lt;input type=\"hidden\" name=\"col\" value=\"priority\"/&gt;\n&lt;input type=\"hidden\" name=\"col\" value=\"component\"/&gt;\n--&gt;\n&lt;/div&gt;&lt;/form&gt;\n}}}\n</pre></td><td style=\"border: none; padding: 1em\" valign=\"top\"><form action=\"/query\" method=\"get\"><div>\n<input type=\"text\" name=\"keywords\" value=\"~\" size=\"30\"/>\n<input type=\"submit\" value=\"Search by Keywords\"/>\n<!-- To control what fields show up use hidden fields\n<input type=\"hidden\" name=\"col\" value=\"id\"/>\n<input type=\"hidden\" name=\"col\" value=\"summary\"/>\n<input type=\"hidden\" name=\"col\" value=\"status\"/>\n<input type=\"hidden\" name=\"col\" value=\"milestone\"/>\n<input type=\"hidden\" name=\"col\" value=\"version\"/>\n<input type=\"hidden\" name=\"col\" value=\"owner\"/>\n<input type=\"hidden\" name=\"col\" value=\"priority\"/>\n<input type=\"hidden\" name=\"col\" value=\"component\"/>\n-->\n</div></form>\n</td></tr></table>\n<h2 id=\"AvailableProcessors\">Available Processors</h2>\n<p>\nThe following processors are included in the Trac distribution:\n</p>\n<dl class=\"wiki\"><dt><tt>#!default</tt></dt><dd>Present the text verbatim in a preformatted text block. \nThis is the same as specifying <em>no</em> processor name\n(and no <tt>#!</tt>)\n</dd><dt><tt>#!comment</tt></dt><dd>Do not process the text in this section (i.e. contents exist\nonly in the plain text - not in the rendered page).\n</dd></dl>\n<h3 id=\"HTMLrelated\">HTML related</h3>\n<dl class=\"wiki\"><dt><tt>#!html</tt></dt><dd>Insert custom HTML in a wiki page.\n</dd><dt><tt>#!htmlcomment</tt></dt><dd>Insert an HTML comment in a wiki page (<em>since 0.12</em>).\n</dd></dl>\n<p>\nNote that <tt>#!html</tt> blocks have to be <em>self-contained</em>,\ni.e. you can't start an HTML element in one block and close it later in a second block. Use the following processors for achieving a similar effect. \n</p>\n<dl class=\"wiki\"><dt><tt>#!div</tt></dt><dd>Wrap an arbitrary Wiki content inside a &lt;div&gt; element\n(<em>since 0.11</em>).\n</dd><dt><tt>#!span</tt></dt><dd>Wrap an arbitrary Wiki content inside a &lt;span&gt; element \n(<em>since 0.11</em>). \n</dd></dl>\n<dl class=\"wiki\"><dt><tt>#!td</tt></dt><dd>Wrap an arbitrary Wiki content inside a &lt;td&gt; element (<em>since 0.12</em>)\n</dd><dt><tt>#!th</tt></dt><dd>Wrap an arbitrary Wiki content inside a &lt;th&gt; element (<em>since 0.12</em>) \n</dd><dt><tt>#!tr</tt></dt><dd>Can optionally be used for wrapping <tt>#!td</tt> and <tt>#!th</tt> blocks,\neither for specifying row attributes of better visual grouping\n(<em>since 0.12</em>)\n</dd></dl>\n<p>\nSee <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/WikiHtml\">WikiHtml</a> for example usage and more details about these processors.\n</p>\n<h3 id=\"OtherMarkups\">Other Markups</h3>\n<dl class=\"wiki\"><dt><tt>#!rst</tt></dt><dd>Trac support for Restructured Text. See <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/WikiRestructuredText\">WikiRestructuredText</a>.\n</dd><dt><tt>#!textile</tt></dt><dd>Supported if <a class=\"ext-link\" href=\"http://cheeseshop.python.org/pypi/textile\"><span class=\"icon\">\u200b</span>Textile</a> \nis installed. \nSee <a class=\"ext-link\" href=\"http://www.textism.com/tools/textile/\"><span class=\"icon\">\u200b</span>a Textile reference</a>.\n</dd></dl>\n<h3 id=\"CodeHighlightingSupport\">Code Highlighting Support</h3>\n<p>\nTrac includes processors to provide inline syntax highlighting:\n</p>\n<blockquote>\n<p>\n<tt>#!c</tt> (C), <tt>#!cpp</tt> (C++), <tt>#!python</tt> (Python), <tt>#!perl</tt> (Perl), \n<tt>#!ruby</tt> (Ruby), <tt>#!php</tt> (PHP), <tt>#!asp</tt> (ASP), <tt>#!java</tt> (Java), \n<tt>#!js</tt> (Javascript), <tt>#!sql (SQL)</tt>, <tt>#!xml</tt> (XML or HTML),\n<tt>#!sh</tt> (Bourne/Bash shell), etc.\n</p>\n</blockquote>\n<p>\nTrac relies on external software packages for syntax coloring,\nlike <a class=\"ext-link\" href=\"http://pygments.org\"><span class=\"icon\">\u200b</span>Pygments</a>. \n</p>\n<p>\nSee <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracSyntaxColoring\">TracSyntaxColoring</a> for information about which languages\nare supported and how to enable support for more languages.\n</p>\n<p>\nNote also that by using the MIME type as processor, it is possible to syntax-highlight the same languages that are supported when browsing source code. For example, you can write:\n</p>\n<pre class=\"wiki\">{{{\n#!text/html\n&lt;h1&gt;text&lt;/h1&gt;\n}}}\n</pre><p>\nThe result will be syntax highlighted HTML code:\n</p>\n<div class=\"code\"><pre><span class=\"nt\">&lt;h1&gt;</span>text<span class=\"nt\">&lt;/h1&gt;</span>\n</pre></div><p>\nThe same is valid for all other <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracSyntaxColoring#SyntaxColoringSupport\">mime types supported</a>.\n</p>\n<p>\nFor more processor macros developed and/or contributed by users, visit:
