Data Characteristics \n<!----------------------------------------------------------------------->\n<H2> Data Characteristics</H2>\nThe HTML source of a web page is given.  Users looked at\neach web page and inidated on a 3 point scale (hot medium cold)\n50-100 pages per domain.  However, this is realistic because we want to\nlearn user profiles from as few examples as possible so that users have\nan incentitive to rate pages.\n\n<!------------------------------------------------------------------------\n<!  Data Format \n<!----------------------------------------------------------------------->\n<H2>Data Format</H2>\n<P>Each subject is in a seperate directory. Within each directory, there\nis an file named \"index\".  The index contains information on the other\nfiles. Each entry is a line of the form:\n</P>\n<PRE> file-name  |  rating  |  url  |  date-rated  |  title </PRE>\n\n<P>\nwhere file-name is the name of a file (usually an integer), rating is hot, \nmedium, or cold.  There are so few medium's that mediums are usually merged \nwith cold in experiments.\n</P>\n\n<P>\nThe other fields aren't used in learning, but they are collected by the \ninterface for other purposes. They are the url of the html source, the date \nrated and the title of the web oage.\n</P>\n\n<!------------------------------------------------------------------------\n<!
