class=\"False\"><a class=\"False\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracNotification\">Notification</a></li></ul></div><p>\n</p>\n<p>\nTrac has a built-in functionality for visualizing \u201cdiffs\u201d - changes to files.\n</p>\n<p>\nThere are different kinds of <em>change sets</em>. \nSome can correspond to revisions made in the repositories,\nothers can aggregate changes made in several revisions, \nbut in the end, any kind of differences can be shown.\n</p>\n<p>\nThe changeset view consists of two parts, the <em>header</em> \nand the <em>diff views</em>.\n</p>\n<h2 id=\"ChangesetHeader\">Changeset Header</h2>\n<p>\nThe header shows an overview of the whole changeset.\nHere you will find information such as:\n</p>\n<ul><li>Timestamp \u2014 When the changeset was commited\n</li><li>Author \u2014 Who commited the changeset\n</li><li>Message \u2014 A brief description from the author (the commit log message)\n</li><li>Location \u2014 Parent directory of all files affected by this changeset\n</li><li>Files \u2014 A list of files affected by this changeset\n</li></ul><p>\nIf more than one revision is involved in the set of changes being\ndisplayed, the <em>Timestamp</em>, <em>Author</em> and <em>Message</em> fields \nwon't be shown.\n</p>\n<p>\nIn front of each listed file, you'll find  a colored rectangle. The color\nindicates how the file is affected by the changeset.\n \n</p>\n<ul><li>Green: Added\n</li><li>Red: Removed\n</li><li>Yellow: Modified\n</li><li>Blue: Copied\n</li><li>Gray: Moved\n</li></ul><p>\nThe color legend is located below the header as a reminder.\n</p>\n<h2 id=\"DiffViews\">Diff Views</h2>\n<p>\nBelow the header is the main part of the changeset, the diff view. Each file is shown in a separate section, each of which will contain only the regions of the file that are affected by the changeset. There are two different styles of displaying the diffs: <em>inline</em> or <em>side-by-side</em> (you can switch between those styles using the preferences form):\n</p>\n<ul><li>The <em>inline</em> style shows the changed regions of a file underneath each other. A region removed from the file will be colored red, an added region will be colored green. If a region was modified, the old version is displayed above the new version. Line numbers on the left side indicate the exact position of the change in both the old and the new version of the file.\n</li><li>The <em>side-by-side</em> style shows the old version on the left and the new version on the right (this will typically require more screen width than the inline style.) Added and removed regions will be colored in the same way as with the inline style (green and red, respectively), but modified regions will have a yellow background.\n</li></ul><p>\nIn addition, various advanced options are available in the preferences form for adjusting the display of the diffs:\n</p>\n<ul><li>You can set how many lines are displayed before and after every change\n(if the value <em>all</em> is used, then the full file will be shown)\n</li><li>You can toggle whether blank lines, case changes and white space changes are ignored, thereby letting you find the functional changes more quickly\n</li></ul><h2 id=\"TheDifferentWaystoGetaDiff\">The Different Ways to Get a Diff</h2>\n<h3 id=\"ExaminingaChangeset\">Examining a Changeset</h3>\n<p>\nWhen viewing a repository check-in, such as when following a\nchangeset <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracLinks\">link</a> or a changeset event in the \n<a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracTimeline\">timeline</a>, Trac will display the exact changes\nmade by the check-in.\n</p>\n<p>\nThere will be also navigation links to the <em>Previous Changeset</em>\nto and <em>Next Changeset</em>.\n</p>\n<h3 id=\"ExaminingDifferencesBetweenRevisions\">Examining Differences Between Revisions</h3>\n<p>\nOften you'll want to look at changes made on a file \nor on a directory spanning multiple revisions. The easiest way\nto get there is from the <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracRevisionLog\">TracRevisionLog</a>, where you can select\nthe <em>old</em> and the <em>new</em> revisions of the file or directory, and\nthen click the <em>View changes</em> button.\n</p>\n<h3 id=\"ExaminingDifferencesBetweenBranches\">Examining Differences Between Branches</h3>\n<p>\nOne of the core features of version control systems is the possibility\nto work simultaneously on different <em>Lines of Developments</em>, commonly\ncalled \u201cbranches\u201d. Trac enables you to examine the exact differences\nbetween such branches.\n</p>\n<p>\nUsing the <strong>View changes ...</strong> button in the <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracBrowser\">TracBrowser</a> allows you to enter\n<em>From:</em> and <em>To:</em> path/revision pairs. The resulting set of differences consist\nof the changes that should be applied to the <em>From:</em> content in order\nto get to the <em>To:</em> content.\n</p>\n<p>\nFor convenience, it is possible to invert the roles of the <em>old</em> and the <em>new</em>\npath/revision pairs by clicking the <em>Reverse Diff</em> link on the changeset page.\n</p>\n<h3 id=\"CheckingtheLastChange\">Checking the Last Change</h3>\n<p>\nThe last possibility for examining changes is to use the <em>Last Change</em>\nlink provided by the <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracBrowser\">TracBrowser</a>.\n</p>\n<p>\nThis link will take you to the last change that was made on that path.\nFrom there, you can use the <em>Previous Change</em> and <em>Next Change</em> links\nto traverse the change history of the file or directory.\n</p>\n<hr />\n<p>\nSee also: <a class=\"wiki\"
