class=\"False\"><a class=\"False\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracNotification\">Notification</a></li></ul></div><p>\n</p>\n<p>\nTrac configuration is done by editing the <strong><tt>trac.ini</tt></strong> config file, located in <tt>&lt;projectenv&gt;/conf/trac.ini</tt>.  Changes to the configuration are usually reflected immediately, though changes to the <tt>[components]</tt> or <tt>[logging]</tt> sections will require restarting the web server. You may also need to restart the web server after creating a global configuration file when none was previously present.\n</p>\n<p>\nThe <tt>trac.ini</tt> configuration file should be writable by the web server, as Trac currently relies on the possibility to trigger a complete environment reload to flush its caches.\n</p>\n<h2 id=\"GlobalConfiguration\">Global Configuration</h2>\n<p>\nIn versions prior to 0.11, the global configuration was by default located in <tt>$prefix/share/trac/conf/trac.ini</tt> or /etc/trac/trac.ini, depending on the distribution. If you're upgrading, you may want to specify that file to inherit from.  Literally, when you're upgrading to 0.11, you have to add an <tt>[inherit]</tt> section to your project's <tt>trac.ini</tt> file. Additionally, you have to move your customized templates and common images from <tt>$prefix/share/trac/...</tt> to the new location.\n</p>\n<p>\nGlobal options will be merged with the environment-specific options, where local options override global options. The options file is specified as follows:\n</p>\n<pre class=\"wiki\">[inherit]\nfile = /path/to/global/trac.ini\n</pre><p>\nMultiple files can be specified using a comma-separated list.\n</p>\n<p>\nNote that you can also specify a global option file when creating a new project,  by adding the option <tt>--inherit=/path/to/global/trac.ini</tt> to <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracAdmin#initenv\">trac-admin</a>'s <tt>initenv</tt> command.  If you do not do this but nevertheless intend to use a global option file with your new environment, you will have to go through the newly generated <tt>conf/trac.ini</tt> file and delete the entries that will otherwise override those set in the global file.\n</p>\n<p>\nThere are two more entries in the <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracIni#inherit-section\"> [inherit] </a> section, <tt>templates_dir</tt> for sharing global templates and <tt>plugins_dir</tt>, for sharing plugins. Those entries can themselves be specified in the shared configuration file, and in fact, configuration files can even be chained if you specify another <tt>[inherit] file</tt> there.\n</p>\n<p>\nNote that the templates found in the <tt>templates/</tt> directory of the <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracEnvironment\">TracEnvironment</a> have precedence over those found in <tt>[inherit] templates_dir</tt>. In turn, the latter have precedence over the installed templates, so be careful about what you put there, notably if you override a default template be sure to refresh your modifications when you upgrade to a new version of Trac (the preferred way to perform <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracInterfaceCustomization\">TracInterfaceCustomization</a> being still to write a custom plugin doing an appropriate <tt>ITemplateStreamFilter</tt> transformation).\n</p>\n<h2 id=\"Referenceforsettings\">Reference for settings</h2>\n<p>\nThis is a brief reference of available configuration options, and their default settings.\n</p>\n<p>\n</p><div class=\"tracini\"><h3 id=\"attachment-section\"><code>[attachment]</code></h3><table class=\"wiki\"><tbody><tr><td><tt>max_size</tt></td><td>Maximum allowed file size (in bytes) for ticket and wiki\nattachments.</td><td class=\"default\"><code>262144</code></td></tr><tr><td><tt>max_zip_size</tt></td><td>Maximum allowed total size (in bytes) for an attachment list to be\ndownloadable as a <tt>.zip</tt>. id=\"components-section\"><code>[components]</code></h3><p>\nThis section is used to enable or disable components\nprovided by plugins, as well as by Trac itself. The component\nto enable/disable is specified via the name of the\noption. Whether its enabled is determined by the option value;\nsetting the value to <tt>enabled</tt> or <tt>on</tt> will enable the\ncomponent, any other value (typically <tt>disabled</tt> or <tt>off</tt>)\nwill disable the component.\n</p>\n<p>\nThe option name is either the fully qualified name of the\ncomponents or the module/package prefix of the component. The\nformer enables/disables a specific component, while the latter\nenables/disables any component in the specified\npackage/module.\n</p>\n<p>\nConsider the following configuration snippet:\n</p>\n<pre class=\"wiki\">[components]\ntrac.ticket.report.ReportModule = disabled\nwebadmin.* = enabled\n</pre><p>\nThe first option tells Trac to disable the\n<a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracReports\">report module</a>.\nThe second option instructs Trac to enable all components in\nthe <tt>webadmin</tt> package. Note that the trailing wildcard is\nrequired for module/package matching.\n</p>\n<p>\nTo view the list of active components, go to the <em>Plugins</em>\npage on <em>About Trac</em> (requires <tt>CONFIG_VIEW</tt>\n<a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracPermissions\">permissions</a>).\n</p>\n<p>\nSee also: <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracPlugins\">TracPlugins</a>\n</p>\n<table class=\"wiki\"><tbody></tbody></table><h3 id=\"header_logo-section\"><code>[header_logo]</code></h3><table id=\"intertrac-section\"><code>[intertrac]</code></h3><p>\nThis section configures <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/InterTrac\">InterTrac</a> prefixes. Options in this section\nwhose name contain a \".\" define aspects of the <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/InterTrac\">InterTrac</a> prefix\ncorresponding to the option name up to the \".\". Options whose name\ndon't contain a \".\" define an alias.\n</p>\n<p>\nThe <tt>.url</tt> is mandatory and is used for locating the other Trac.\nThis can be a relative URL in case that Trac environment is located\non the same server.\n</p>\n<p>\nThe <tt>.title</tt> information is used for providing a useful tooltip when\nmoving the cursor over an <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/InterTrac\">InterTrac</a> link.\n</p>\n<p>\nThe <tt>.compat</tt> option can be used to activate or disable a\n<em>compatibility</em> mode:\n</p>\n<ul><li>If the targeted Trac is running a version below\n<a class=\"ext-link\" href=\"http://trac.edgewall.org/intertrac/milestone%3A0.10\" title=\"milestone:0.10 in Trac project trac\"><span class=\"icon\">\u200b</span>0.10</a> (<a class=\"ext-link\" href=\"http://trac.edgewall.org/intertrac/r3526\" title=\"r3526 in Trac project trac\"><span class=\"icon\">\u200b</span>r3526</a> to be precise), then\nit doesn't know how to dispatch an <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/InterTrac\">InterTrac</a> link, and it's up to\nthe local Trac to prepare the correct link. Not all links will work\nthat way, but the most common do. This is called the compatibility\nmode, and is <tt>true</tt> by default.\n</li><li>If you know that the remote Trac knows how to dispatch <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/InterTrac\">InterTrac</a>\nlinks, you can explicitly disable this compatibility mode and then\n<em>any</em> <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracLinks\">TracLinks</a> can become <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/InterTrac\">InterTrac</a> links.\n</li></ul><p>\nExample configuration:\n</p>\n<pre class=\"wiki\">[intertrac]\n# -- Example of setting up an alias:\nt = trac\n\n# -- Link to an external Trac:\ntrac.title = Edgewall's Trac for Trac\ntrac.url = http://trac.edgewall.org\n</pre><table class=\"wiki\"><tbody></tbody></table><h3 id=\"interwiki-section\"><code>[interwiki]</code></h3><p>\nEvery option in the <tt>[interwiki]</tt> section defines one <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/InterWiki\">InterWiki</a>\nprefix. The option name defines the prefix. The option value defines\nthe URL, optionally followed by a description separated from the URL\nby whitespace. The component to enable/disable is specified via the name of the option. Whether its enabled is determined by the option value; setting the value to <tt>enabled</tt> or <tt>on</tt> will enable the component, any other value (typically <tt>disabled</tt> or <tt>off</tt>) will disable the component.\n</p>\n<p>\nThe option name is either the fully qualified name of the components or the module/package prefix of the component. The former enables/disables a specific component, while the latter enables/disables any component in the specified package/module.\n</p>\n<p>\nConsider the following configuration snippet:\n</p>\n<pre class=\"wiki\">[components]\ntrac.ticket.report.ReportModule = disabled\nwebadmin.* = enabled\n</pre><p>\nThe first option tells Trac to disable the <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracReports\">report module</a>. The second option instructs Trac to enable all components in the <tt>webadmin</tt> package. Note that the trailing wildcard is required for module/package matching.\n</p>\n<p>\nSee the <em>Plugins</em> page on <em>About Trac</em> to get the list of active components (requires <tt>CONFIG_VIEW</tt> <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracPermissions\">permissions</a>.)\n</p>\n<p>\nSee also: <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracPlugins\">TracPlugins</a>\n</p>\n<h3 id=\"milestone-groups-section\">[milestone-groups]</h3>\n<p>\n<em>(since 0.11)</em>\n</p>\n<p>\nAs the workflow for tickets is now configurable, there can be many ticket states,\nand simply displaying closed tickets vs. all the others is maybe not appropriate \nin all cases. This section enables one to easily create <em>groups</em> of states \nthat will be shown in different colors in the milestone progress bar.\n</p>\n<p>\nExample configuration (the default only has closed and active):\n</p>\n<pre class=\"wiki\">closed = closed\n# sequence number in the progress bar\nclosed.order = 0\n# optional extra param for the query (two additional columns: created and modified and sort on created)\nclosed.query_args = group=resolution,order=time,col=id,col=summary,col=owner,col=type,col=priority,col=component,col=severity,col=time,col=changetime\n# indicates groups that count for overall completion percentage\nclosed.overall_completion = true\n\nnew = new\nnew.order = 1\nnew.css_class = new\nnew.label = new\n\n# one catch-all group is allowed\nactive = *\nactive.order = 2\n# CSS class for this interval\nactive.css_class = open\n# Displayed label for this group\nactive.label = in progress\n</pre><p>\nThe definition consists in a comma-separated list of accepted status.\nAlso, '*' means any status and could be used to associate all remaining\nstates to one catch-all group.\n</p>\n<p>\nThe CSS class can be one of: new (yellow), open (no color) or\nclosed (green). New styles can easily be added using the following\nselector:  <tt>table.progress td.&lt;class&gt;</tt>\n</p>\n<h3 id=\"repositories-section\">[repositories]</h3>\n<p>\n(<em>since 0.12</em> alternatives.\n</p>\n<h3 id=\"svn:externals-section\">[svn:externals]</h3>\n<p>\n<em>(since 0.11)</em>\n</p>\n<p>\nThe <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracBrowser\">TracBrowser</a> for Subversion can interpret the <tt>svn:externals</tt> property of folders.\nBy default, it only turns the URLs into links as Trac can't browse remote repositories.\n</p>\n<p>\nHowever, if you have another Trac instance (or an other repository browser like <a class=\"ext-link\" href=\"http://www.viewvc.org/\"><span class=\"icon\">\u200b</span>ViewVC</a>) configured to browse the target repository, then you can instruct Trac which other repository browser to use for which external URL.\n</p>\n<p>\nThis mapping is done in the <tt>[svn:externals]</tt> section of the <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracIni\">TracIni</a>\n</p>\n<p>\nExample:\n</p>\n<pre class=\"wiki\">[svn:externals]\n1 = svn://server/repos1                       http://trac/proj1/browser/$path?rev=$rev\n2 = svn://server/repos2                       http://trac/proj2/browser/$path?rev=$rev\n3 = http://theirserver.org/svn/eng-soft       http://ourserver/viewvc/svn/$path/?pathrev=25914\n4 = svn://anotherserver.com/tools_repository  http://ourserver/tracs/tools/browser/$path?rev=$rev\n</pre><p>\nWith the above, the <tt>svn://anotherserver.com/tools_repository/tags/1.1/tools</tt> external will be mapped to <tt>http://ourserver/tracs/tools/browser/tags/1.1/tools?rev=</tt> (and <tt>rev</tt> will be set to the appropriate revision number if the external additionally specifies a revision, see the <a class=\"ext-link\" href=\"http://svnbook.red-bean.com/en/1.4/svn.advanced.externals.html\"><span class=\"icon\">\u200b</span>SVN Book on externals</a> for more details).\n</p>\n<p>\nNote that the number used as a key in the above section is purely used as a place holder, as the URLs themselves can't be used as a key due to various limitations in the configuration file parser.\n</p>\n<p>\nFinally, the relative URLs introduced in <a class=\"ext-link\" href=\"http://subversion.tigris.org/svn_1.5_releasenotes.html#externals\"><span class=\"icon\">\u200b</span>Subversion 1.5</a> are not yet supported.\n</p>\n<h3 id=\"ticket-custom-section\">[ticket-custom]</h3>\n<p>\nIn this section, you can define additional fields for tickets.
