Abstract \n<!----------------------------------------------------------------------->\n<H2>Abstract</H2>\n<p>\nData giving characteristics of each ORF (potential gene) in the\nM. tuberculosis bacterium. Sequence, homology (similarity to other\ngenes) and structural information, and function (if known) are provided.\n</p>\n\n<!------------------------------------------------------------------------\n<!  Sources \n<!----------------------------------------------------------------------->\n<H2> Sources</H2>\n<H4> Original Owner and Donor</H4>\n<PRE>\n  <a href=\"http://users.aber.ac.uk/rdk\">Ross D. Data Characteristics\n<!----------------------------------------------------------------------->\n<H2> Data Characteristics</H2>\n<P>\nThe data was collected from several sources, including the <a href=\"http://www.sanger.ac.uk/Projects/M_tuberculosis/Gene_list/\">Sanger Centre</a> and <a href=\"http://www.expasy.ch/sprot/sprot-top.html\">SWISSPROT</a>.  Structure prediction was made by <a href=\"http://www.aber.ac.uk/~phiwww/prof/index.html\">PROF</a>. Homology search was made by <a href=\"http://www.ncbi.nlm.nih.gov/BLAST/\">PSI-BLAST</a>. \n\n<p>\nThe data is in Datalog format. Missing values are not explicit, but some \ngenes have more relationships than others. \n\n\n<H4>Dependencies</H4>\n\n<p>\nM. tuberculosis genes (ORFs) are related to each other by the predicate \ntb_to_tb_evalue(TBNumber,E-value). \nThey are related to other (SWISSPROT) proteins by the predicate\ne_val(AccNo,E-value).\nAll the data for a single gene (ORF) is enclosed between delimiters of \nthe form:\n<pre>\n   begin(model(TBNumber)).\n   end(model(TBNumber)).\n</pre>\n\n\n\n\n<!------------------------------------------------------------------------\n<!  Other Relevant Information \n<!----------------------------------------------------------------------->\n<H2> Other Relevant Information</H2>\n\n<p>\nThe gene functional classes are in a hierarchy. See\n<a href=\"http://www.sanger.ac.uk/Projects/M_tuberculosis/Gene_list/\">http://www.sanger.ac.uk/Projects/M_tuberculosis/Gene_list/</a>.\n</p>\n\n\n<!------------------------------------------------------------------------\n<!  Data Format  \n<!----------------------------------------------------------------------->\n<H2>Data Format</H2>\nThere are two datalog files: tb_data.pl and ecoli_functions.pl\n\n<h4>tb_functions.pl</h4>\n\nLists classes and ORF functions. Lines are of the following form:\n\n<PRE>\n   class([1,0,0,0],\"Small-molecule metabolism \").\n   class([1,1,0,0],\"Degradation \").\n   class([1,1,1,0],\"Carbon compounds \").  \n</PRE>\n\n<p>\nArguments are a list of 4 numbers (describing class at the 4\ndifferent levels), followed by a string class description. For example,\n\n<pre>\n   function(tb186,[1,1,1,0],'bglS',\"beta-glucosidase\"). \n</pre>\n\n<p>\nArguments are ORF number, list of 4 class numbers, gene name (or\nnull if no gene name) in single quotes, ORF description in double quotes.\n\n\n<h4>tb_data.pl</h4>\nData for each ORF (gene) is delimited by \n<pre>\n   begin(model(X)).\n   end(model(X)).\n</pre>\nwhere X is the ORF number. Other predicates are as follows (examples):\n\n<PRE>\n   tb_protein(X).    % X is gene number\n   function(2,1,5,0,'gyrA','DNA gyrase subunit A').  % 4 levels of functional hierarchy, gene name, description\n   coding_region(7302,9815). % start,end. integers\n   tb_mol_wt(19934).  % integer\n   access(1,e,20). % int (position), {e,i,b}, int (length) \n   access_exposed(1,20). % int (position), int (length) \n   access_intermediate(26,1). % int (position), int (length) \n   access_burried(1,2). % int (position), int (length) \n   access_dist(b,42.8). % {e,i,b}, float (percentage)\n   sec_struc(1,c,23). % int (position), {a,b,c}, int (length)\n   sec_struc_coil(1,23). % int (position), int (length)\n   sec_struc_alpha(1,15). % int (position), int (length)\n   sec_struc_beta(1,6). % int (position), int (length)\n   struc_dist(a,32.1). % {a,b,c}, float (percentage)\n   sec_struc_conf(78.8). % float (confidence)\n   sec_struc_conf_alpha(88.9). % float (confidence)\n   sec_struc_conf_beta(58.0). % float (confidence)\n   sec_struc_conf_coil(77.7). % float (confidence)\n   psi_sequences_found(1,7). % how many found, which iteration\n   psi_sequences_found_again(2,7).  % how many found, which iteration\n   psi_sequences_found_new(2,0). % how many found, which iteration\n   amino_acid_ratio(a,11.2). % amino acid letter, float\n   amino_acid_pair_ratio(a,c,0.0). % amino acid letter, amino acid letter, float (out of 1000, ie 2.8 = 0.28%)\n   sequence_length(187).  % integer\n   tb_to_tb_evalue(tb3671,1.100000e-01). % ORF number, e-value (double)  \n   e_val(p35925,7.0e-59). % SWISSPROT accession no, e-value (double)\n   species(p35925,'streptomyces_coelicolor'). % SWISSPROT acc no, string\n   classification(p35925,bacteria). % SWISSPROT acc no, name\n   mol_wt(p35925,19772). % SWISSPROT acc no, integer\n   keyword(p35925,'hypothetical_protein'). % SWISSPROT acc no, string\n   db_ref(p35925,embl,l27063,g436026,null). % SWISSPROT acc no, db id, primary id, secondary id, status id\n   signalip(c,35,no). % {c,y,s}, int (signal peptide c/y/s score), yes/no\n   signalip(ss,1,34,no). % ss, int, int, yes/no\n   signalip(cleavage,59,60). % cleavage, int/null, int/null\n   hydro_cons(-0.498,-0.474,0.624,3.248,0.278). % double, double, double, double, double\n   gene_name(p41514,'gyrb'). % SWISSPROT acc no, string\n</PRE>\n\n\n\n\n\n\n<!------------------------------------------------------------------------\n<!
