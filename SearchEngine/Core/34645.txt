class=\"wikipage searchable\">\n        \n          <div id=\"wikipage\" class=\"trac-content\"><h1 id=\"Importingticketdata\">Importing ticket data</h1>\n<h2 id=\"Bugzilla\">Bugzilla</h2>\n<p>\nTicket data can be imported from Bugzilla using the <a class=\"ext-link\" href=\"http://trac.edgewall.org/browser/trunk/contrib/bugzilla2trac.py\"><span class=\"icon\">\u200b</span>bugzilla2trac.py</a> script, available in the contrib/ directory of the Trac distribution.\n</p>\n<pre class=\"wiki\">$ bugzilla2trac.py\nbugzilla2trac - Imports a bug database from Bugzilla into Trac.\n\nUsage: bugzilla2trac.py [options]\n\nAvailable Options:\n  --db &lt;MySQL dbname&gt;              - Bugzilla's database\n  --tracenv /path/to/trac/env      - full path to Trac db environment\n  -h | --host &lt;MySQL hostname&gt;     - Bugzilla's DNS host name\n  -u | --user &lt;MySQL username&gt;     - effective Bugzilla's database user\n  -p | --passwd &lt;MySQL password&gt;   - Bugzilla's user password\n  -c | --clean                     - remove current Trac tickets before importing\n  --help | help                    - this help info\n\nAdditional configuration options can be defined directly in the script.\n</pre><p>\nCurrently, the following data is imported from Bugzilla:\n</p>\n<ul><li>bugs\n</li><li>bug activity (field changes)\n</li><li>bug attachments\n</li><li>user names and passwords (put into a htpasswd file)\n</li></ul><p>\nThe script provides a number of features to ease the conversion, such as:\n</p>\n<ul><li>PRODUCT_KEYWORDS:  Trac doesn't have the concept of products, so the script provides the ability to attach a ticket keyword instead.\n</li></ul><ul><li>IGNORE_COMMENTS:  Don't import Bugzilla comments that match a certain regexp.\n</li></ul><ul><li>STATUS_KEYWORDS:  Attach ticket keywords for the Bugzilla statuses not available in Trac.  By default, the 'VERIFIED' and 'RELEASED' Bugzilla statuses are translated into Trac keywords.\n</li></ul><p>\nFor more details on the available options, see the configuration section at the top of the script.\n</p>\n<h2 id=\"Sourceforge\">Sourceforge</h2>\n<p>\nTicket data can be imported from Sourceforge using the <a class=\"ext-link\" href=\"http://trac.edgewall.org/browser/trunk/contrib/sourceforge2trac.py\"><span class=\"icon\">\u200b</span>sourceforge2trac.py</a> script, available in the contrib/ directory of the Trac distribution.\n</p>\n<p>\nSee #Trac3521 for an updated sourceforge2trac script.\n</p>\n<h2 id=\"Mantis\">Mantis</h2>\n<p>\nThe mantis2trac script now lives at <a class=\"ext-link\" href=\"http://trac-hacks.org/wiki/MantisImportScript\"><span class=\"icon\">\u200b</span>http://trac-hacks.org/wiki/MantisImportScript</a>. You can always get the latest version from <a class=\"ext-link\" href=\"http://trac-hacks.org/changeset/latest/mantisimportscript?old_path=/&amp;filename=mantisimportscript&amp;format=zip\"><span class=\"icon\">\u200b</span>http://trac-hacks.org/changeset/latest/mantisimportscript?old_path=/&amp;filename=mantisimportscript&amp;format=zip</a>\n</p>\n<p>\nMantis bugs can be imported using the attached script.\n</p>\n<p>\nCurrently, the following data is imported from Mantis:\n</p>\n<ul><li>bugs\n</li><li>bug comments\n</li><li>bug activity (field changes)\n</li><li>attachments (as long as the files live in the mantis db, not on the filesystem) \n</li></ul><p>\nIf you use the script, please read the NOTES section (at the top of the file) and make sure you adjust the config parameters for your environment.\n</p>\n<p>\nmantis2trac.py has the same parameters as the bugzilla2trac.py script:\n</p>\n<pre class=\"wiki\">mantis2trac - Imports a bug database from Mantis into Trac.\n\nUsage: mantis2trac.py [options] \n\nAvailable Options:\n  --db &lt;MySQL dbname&gt;              - Mantis database\n  --tracenv /path/to/trac/env      - Full path to Trac db environment\n  -h | --host &lt;MySQL hostname&gt;     - Mantis DNS host name\n  -u | --user &lt;MySQL username&gt;     - Effective Mantis database user\n  -p | --passwd &lt;MySQL password&gt;   - Mantis database user password\n  -c | --clean                     - Remove current Trac tickets before importing\n  --help | help                    - This help info\n\nAdditional configuration options can be defined directly in the script.\n</pre><h2 id=\"Jira\">Jira</h2>\n<p>\nThe <a class=\"ext-link\" href=\"http://trac-hacks.org/wiki/JiraToTracIntegration\"><span class=\"icon\">\u200b</span>Jira2Trac plugin</a> provides you with tools to import Atlassian Jira backup files into Trac.\n</p>\n<p>\nThe plugin consists of a Python 3.1 commandline tool that:\n</p>\n<ul><li>Parses the Jira backup XML file\n</li><li>Sends the imported Jira data and attachments to Trac using the <a class=\"ext-link\" href=\"http://trac-hacks.org/wiki/XmlRpcPlugin\"><span class=\"icon\">\u200b</span>XmlRpcPlugin</a>\n</li><li>Generates a htpasswd file containing the imported Jira users and their SHA-512 base64 encoded passwords\n</li></ul><h2 id=\"Other\">Other</h2>\n<p>\nSince trac uses a SQL database to store the data, you can import from other systems by examining the database tables. Just go into <a class=\"ext-link\" href=\"http://www.sqlite.org/sqlite.html\"><span class=\"icon\">\u200b</span>sqlite</a> command line to look at the tables and import into them from your application.\n</p>\n<h3 id=\"Usingacommadelimitedfile-CSV\">Using a comma delimited file - CSV</h3>\n<p>\nSee <a class=\"ext-link\" href=\"http://trac.edgewall.org/attachment/wiki/TracSynchronize/csv2trac.2.py\"><span class=\"icon\">\u200b</span>http://trac.edgewall.org/attachment/wiki/TracSynchronize/csv2trac.2.py</a> for details.  This approach is particularly useful if one needs to enter a large number of tickets by hand. (note that the ticket type type field, (task etc...) is also needed for this script to work with more recent Trac releases)\nComments on script: The script has an error on line 168, ('Ticket' needs to be 'ticket').  Also, the listed values for severity and priority are swapped. \n</p>\n<h3 id=\"UsinganExcel.xlsorcommadelimitedfile.csv\">Using an Excel (.xls) or comma delimited file (.csv)</h3>\n<p>\nThis plugin <a class=\"ext-link\" href=\"http://trac-hacks.org/wiki/TicketImportPlugin\"><span class=\"icon\">\u200b</span>http://trac-hacks.org/wiki/TicketImportPlugin</a> lets you import into Trac a series of tickets from a CSV file or (if the xlrd library is installed) from an Excel file.\n</p>\n<p>\nYou can also use it to modify tickets in batch, by saving a report as CSV, editing the CSV file, and re-importing the tickets.\n</p>\n<p>\nThis plugin is very useful when starting a new project: you can import a list of requirements that may have come from meeting notes, list of features, other ticketing systems... It's also great to review the tickets off-line, or to do massive changes to tickets.\n</p>\n<p>\nBased on the ticket id (or, if no id exists, on the summary) in the imported file, tickets are either created or updated. \n</p>\n</div>\n          \n          <div class=\"trac-modifiedby\">\n            <span><a href=\"/LUCICodeRepository/nomaticIM/wiki/TracImport?action=diff&amp;version=3\"
