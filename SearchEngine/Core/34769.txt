id=\"CustomTicketFields\">Custom Ticket Fields</h1>\n<p>\nTrac supports adding custom, user-defined fields to the ticket module. Using custom fields, you can add typed, site-specific properties to tickets.\n</p>\n<h2 id=\"Configuration\">Configuration</h2>\n<p>\nConfiguring custom ticket fields is done in the <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracIni\">trac.ini</a> file. All field definitions should be under a section named <tt>[ticket-custom]</tt>.\n</p>\n<p>\nThe syntax of each field definition is:\n</p>\n<pre class=\"wiki\"> FIELD_NAME = TYPE\n (FIELD_NAME.OPTION = VALUE)\n ...\n</pre><p>\nThe example below should help to explain the syntax.\n</p>\n<h3 id=\"AvailableFieldTypesandOptions\">Available Field Types and Options</h3>\n<ul><li><strong>text</strong>: A simple (one line) text field.\n<ul><li>label: Descriptive label.\n</li><li>value: Default value.\n</li><li>order: Sort order placement. (Determines relative placement in forms with respect to other custom fields.)\n</li><li>format: Either <tt>plain</tt> for plain text or <tt>wiki</tt> to interpret the content as <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/WikiFormatting\">WikiFormatting</a>. (<em>since 0.11.3</em>)\n</li></ul></li><li><strong>checkbox</strong>: A boolean value check box.\n<ul><li>label: Descriptive label.\n</li><li>value: Default value (0 or 1).\n</li><li>order: Sort order placement.\n</li></ul></li><li><strong>select</strong>: Drop-down select box. Uses a list of values.\n<ul><li>label: Descriptive label.\n</li><li>options: List of values, separated by <strong>|</strong> (vertical pipe).\n</li><li>value: Default value (one of the values from options).\n</li><li>order: Sort order placement.\n</li></ul></li><li><strong>radio</strong>: Radio buttons. Essentially the same as <strong>select</strong>.\n<ul><li>label: Descriptive label.\n</li><li>options: List of values, separated by <strong>|</strong> (vertical pipe).\n</li><li>value: Default value (one of the values from options).\n</li><li>order: Sort order placement.\n</li></ul></li><li><strong>textarea</strong>: Multi-line text area.\n<ul><li>label: Descriptive label.\n</li><li>value: Default text.\n</li><li>cols: Width in columns.\n</li><li>rows: Height in lines.\n</li><li>order: Sort order placement.\n</li><li>format: Either <tt>plain</tt> for plain text or <tt>wiki</tt> to interpret the content as <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/WikiFormatting\">WikiFormatting</a>. (<em>since 0.11.3</em>)\n</li></ul></li></ul><h3 id=\"SampleConfig\">Sample Config</h3>\n<pre class=\"wiki\">[ticket-custom]\n\ntest_one = text\ntest_one.label = Just a text box\n\ntest_two = text\ntest_two.label = Another text-box\ntest_two.value = Default [mailto:joe@nospam.com owner]\ntest_two.format = wiki\n\ntest_three = checkbox\ntest_three.label = Some checkbox\ntest_three.value = 1\n\ntest_four = select\ntest_four.label = My selectbox\ntest_four.options = one|two|third option|four\ntest_four.value = two\n\ntest_five = radio\ntest_five.label = Radio buttons are fun\ntest_five.options = uno|dos|tres|cuatro|cinco\ntest_five.value = dos\n\ntest_six = textarea\ntest_six.label = This is a large textarea\ntest_six.value = Default text\ntest_six.cols = 60\ntest_six.rows = 30\n</pre><p>\n<em>Note: To make entering an option for a <tt>select</tt> type field optional, specify a leading <tt>|</tt> in the <tt>fieldname.options</tt> option.</em>\n</p>\n<h3 id=\"ReportsInvolvingCustomFields\">Reports Involving Custom Fields</h3>\n<p>\nCustom ticket fields are stored in the <tt>ticket_custom</tt> table, not in the <tt>ticket</tt> table. So to display the values from custom fields in a report, you will need a join on the 2 tables. Let's use an example with a custom ticket field called <tt>progress</tt>.\n</p>\n<div class=\"code\"><pre><span class=\"k\">SELECT</span> p<span class=\"p\">.</span>value <span class=\"k\">AS</span> __color__<span class=\"p\">,</span>\n   id <span class=\"k\">AS</span> ticket<span class=\"p\">,</span> summary<span class=\"p\">,</span> <span class=\"k\">owner</span><span class=\"p\">,</span> <span class=\"k\">c</span><span class=\"p\">.</span>value <span class=\"k\">AS</span> progress\n  <span class=\"k\">FROM</span> ticket t<span class=\"p\">,</span> enum p<span class=\"p\">,</span> ticket_custom <span class=\"k\">c</span>\n  <span class=\"k\">WHERE</span> status <span class=\"k\">IN</span> <span class=\"p\">(</span><span class=\"s1\">'assigned'</span><span class=\"p\">)</span> <span class=\"k\">AND</span> t<span class=\"p\">.</span>id <span class=\"o\">=</span> <span class=\"k\">c</span><span class=\"p\">.</span>ticket <span class=\"k\">AND</span> <span class=\"k\">c</span><span class=\"p\">.</span>name <span class=\"o\">=</span> <span class=\"s1\">'progress'</span>\n<span class=\"k\">AND</span> p<span class=\"p\">.</span>name <span class=\"o\">=</span> t<span class=\"p\">.</span>priority <span class=\"k\">AND</span> p<span class=\"p\">.</span><span class=\"k\">type</span> <span class=\"o\">=</span> <span class=\"s1\">'priority'</span>\n  <span class=\"k\">ORDER</span> <span class=\"k\">BY</span> p<span class=\"p\">.</span>value\n</pre></div><p>\n<strong>Note</strong> that this will only show tickets that have progress set in them, which is <strong>not the same as showing all tickets</strong>. If you created this custom ticket field <em>after</em> you have already created some tickets, they will not have that field defined, and thus they will never show up on this ticket query. If you go back and modify those tickets, the field will be defined, and they will appear in the query. If that's all you want, you're set.\n</p>\n<p>\nHowever, if you want to show all ticket entries (with progress defined and without), you need to use a <tt>JOIN</tt> for every custom field that is in the query.\n</p>\n<div class=\"code\"><pre><span class=\"k\">SELECT</span> p<span class=\"p\">.</span>value <span class=\"k\">AS</span> __color__<span class=\"p\">,</span>\n   id <span class=\"k\">AS</span> ticket<span class=\"p\">,</span> summary<span class=\"p\">,</span> component<span class=\"p\">,</span> <span class=\"k\">version</span><span class=\"p\">,</span> milestone<span class=\"p\">,</span> severity<span class=\"p\">,</span>\n   <span class=\"p\">(</span><span class=\"k\">CASE</span> status <span class=\"k\">WHEN</span> <span class=\"s1\">'assigned'</span> <span class=\"k\">THEN</span> <span class=\"k\">owner</span><span class=\"o\">||</span><span class=\"s1\">' *'</span> <span class=\"k\">ELSE</span> <span class=\"k\">owner</span> <span class=\"k\">END</span><span class=\"p\">)</span> <span class=\"k\">AS</span> <span class=\"k\">owner</span><span class=\"p\">,</span>\n   time <span class=\"k\">AS</span> created<span class=\"p\">,</span>\n   changetime <span class=\"k\">AS</span> _changetime<span class=\"p\">,</span> description <span class=\"k\">AS</span> _description<span class=\"p\">,</span>\n   reporter <span class=\"k\">AS</span> _reporter<span class=\"p\">,</span>\n  <span class=\"p\">(</span><span class=\"k\">CASE</span> <span class=\"k\">WHEN</span> <span class=\"k\">c</span><span class=\"p\">.</span>value <span class=\"o\">=</span> <span class=\"s1\">'0'</span> <span class=\"k\">THEN</span> <span class=\"s1\">'None'</span> <span class=\"k\">ELSE</span> <span class=\"k\">c</span><span class=\"p\">.</span>value <span class=\"k\">END</span><span class=\"p\">)</span> <span class=\"k\">AS</span> progress\n  <span class=\"k\">FROM</span> ticket t\n     <span class=\"k\">LEFT</span> <span class=\"k\">OUTER</span> <span class=\"k\">JOIN</span> ticket_custom <span class=\"k\">c</span> <span class=\"k\">ON</span> <span class=\"p\">(</span>t<span class=\"p\">.</span>id <span class=\"o\">=</span> <span class=\"k\">c</span><span class=\"p\">.</span>ticket <span class=\"k\">AND</span> <span class=\"k\">c</span><span class=\"p\">.</span>name <span class=\"o\">=</span> <span class=\"s1\">'progress'</span><span class=\"p\">)</span>\n     <span class=\"k\">JOIN</span> enum p <span class=\"k\">ON</span> p<span class=\"p\">.</span>name <span class=\"o\">=</span> t<span class=\"p\">.</span>priority <span class=\"k\">AND</span> p<span class=\"p\">.</span><span class=\"k\">type</span><span class=\"o\">=</span><span class=\"s1\">'priority'</span>\n  <span class=\"k\">WHERE</span> status <span class=\"k\">IN</span> <span class=\"p\">(</span><span class=\"s1\">'new'</span><span class=\"p\">,</span> <span class=\"s1\">'assigned'</span><span class=\"p\">,</span> <span class=\"s1\">'reopened'</span><span class=\"p\">)</span>\n  <span class=\"k\">ORDER</span> <span class=\"k\">BY</span> p<span class=\"p\">.</span>value<span class=\"p\">,</span> milestone<span class=\"p\">,</span> severity<span class=\"p\">,</span> time\n</pre></div><p>\nNote in particular the <tt>LEFT OUTER JOIN</tt> statement here.\n</p>\n<h3 id=\"Updatingthedatabase\">Updating the database</h3>\n<p>\nAs noted above, any tickets created before a custom field has been defined will not have a value for that field. Here's a bit of SQL (tested with SQLite) that you can run directly on the Trac database to set an initial value for custom ticket fields. Inserts the default value of 'None' into a custom field called 'request_source' for all tickets that have no existing value:\n</p>\n<div class=\"code\"><pre><span class=\"k\">INSERT</span> <span class=\"k\">INTO</span> ticket_custom\n   <span class=\"p\">(</span>ticket<span class=\"p\">,</span> name<span class=\"p\">,</span> value<span class=\"p\">)</span>\n   <span class=\"k\">SELECT</span> \n      id <span class=\"k\">AS</span> ticket<span class=\"p\">,</span>\n      <span class=\"s1\">'request_source'</span> <span class=\"k\">AS</span> name<span class=\"p\">,</span>\n      <span class=\"s1\">'None'</span> <span class=\"k\">AS</span> value\n   <span class=\"k\">FROM</span> ticket \n   <span class=\"k\">WHERE</span> id <span class=\"k\">NOT</span> <span class=\"k\">IN</span> <span class=\"p\">(</span>\n      <span class=\"k\">SELECT</span> ticket <span class=\"k\">FROM</span> ticket_custom\n   <span class=\"p\">);</span>\n</pre></div><p>\nIf you added multiple custom fields at different points in time, you should be more specific in the subquery on table <tt>ticket</tt> by adding the exact custom field name to the query:\n</p>\n<div class=\"code\"><pre><span class=\"k\">INSERT</span> <span class=\"k\">INTO</span> ticket_custom\n   <span class=\"p\">(</span>ticket<span class=\"p\">,</span> name<span class=\"p\">,</span> value<span class=\"p\">)</span>\n   <span class=\"k\">SELECT</span> \n      id <span class=\"k\">AS</span> ticket<span class=\"p\">,</span>\n      <span class=\"s1\">'request_source'</span> <span class=\"k\">AS</span> name<span class=\"p\">,</span>\n      <span class=\"s1\">'None'</span> <span class=\"k\">AS</span> value\n   <span class=\"k\">FROM</span> ticket \n   <span class=\"k\">WHERE</span> id <span class=\"k\">NOT</span> <span class=\"k\">IN</span> <span class=\"p\">(</span>\n      <span class=\"k\">SELECT</span> ticket <span class=\"k\">FROM</span> ticket_custom <span class=\"k\">WHERE</span> name <span class=\"o\">=</span> <span class=\"s1\">'request_source'</span>\n   <span class=\"p\">);</span>\n</pre></div><hr />\n<p>\nSee also: <a class=\"wiki\"
