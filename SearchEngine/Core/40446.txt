collection of molecules, calculate the molecular weight \n(atomic units) of each.  Once this is done, create pools of &lt;poolSize&gt; \nmolecules where the molecular weight of the molecules in each pool is \nspaced as far as possible.  Output the molecules back, including \ntheir molecular weight and the index of the pool they were assigned to.\n\nFor example, if there were molecules named (A,B,C,D,E) with weights\n(1,2,3,4,5) and a poolSize = 2, then this should output\n\nA   1   0\nD   4   0\nB   2   1\nE   5   1\nC   3   2\n\nInput: \n- sourceFile:  Molecule file\n    Can be any format understandable by oemolistream, assuming a properly \n    named extension.  For example, &quot;molecules.smi&quot; for SMILES format\n    Can take stdin as source by specifying the filename &quot;-&quot; or &quot;.smi&quot; or \n    something similar.  See documentation of oemolistream for more information.\n\n- poolSize:  Integer\n    Size of the pools to generate.  All pools should be of this size, except for\n    the last one, which might be less.  If set to 0 or a negative number,\n    then assume no pools are to be generated, just output the whole\n    original list with the molecular weights, in order by weight.\n\nOutput:\n- poolFile:  Molecule file\n    Molecules from source output again, but the title / label of the molecules\n    will have appended the molecule's molecular weight and the index of the\n    pool they have been assigned to.  These will be tab-delimited, thus, if\n    the output is simple SMILES, the results should be easily opened\n    in a spreadsheet program (i.e. Excel) additional sorting by pool #, etc.\n\n</pre><br
