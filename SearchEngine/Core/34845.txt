class=\"False\"><a class=\"False\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracNotification\">Notification</a></li></ul></div><p>\n</p>\n<p>\nSince Trac uses a database backend, some extra care is required to safely create a backup of a <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracEnvironment\">project environment</a>. Luckily, <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracAdmin\">trac-admin</a> has a command to make backups easier: <tt>hotcopy</tt>.\n</p>\n<blockquote>\n<p>\n<em>Note: Trac uses the <tt>hotcopy</tt> nomenclature to match that of <a class=\"ext-link\" href=\"http://subversion.tigris.org/\"><span class=\"icon\">\u200b</span>Subversion</a>, to make it easier to remember when managing both Trac and Subversion servers.</em>\n</p>\n</blockquote>\n<h2 id=\"CreatingaBackup\">Creating a Backup</h2>\n<p>\nTo create a backup of a live <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracEnvironment\">TracEnvironment</a>, simply run:\n</p>\n<pre class=\"wiki\">\n  $ trac-admin /path/to/projenv hotcopy /path/to/backupdir\n\n</pre><p>\n<a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracAdmin\">trac-admin</a> will lock the database while copying.<em>\n</em></p>\n<p>\nThe resulting backup directory is safe to handle using standard file-based backup tools like <tt>tar</tt> or <tt>dump</tt>/<tt>restore</tt>.\n</p>\n<p>\nPlease, note, that hotcopy command does not overwrite target directory and when such exists, hotcopy ends with error: <tt>Command failed: [Errno 17] File exists:</tt> This is discussed in <a class=\"ext-link\" href=\"http://trac.edgewall.org/intertrac/ticket%3A3198\" title=\"ticket:3198 in Trac project trac\"><span class=\"icon\">\u200b</span>#3198</a>.\n</p>\n<h3 id=\"RestoringaBackup\">Restoring a Backup</h3>\n<p>\nBackups are simply a copied snapshot of the entire <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracEnvironment\">project environment</a> directory, including the SQLite database. \n</p>\n<p>\nTo restore an environment from a backup, stop the process running Trac (i.e. the Web server or <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracStandalone\">tracd</a>), restore the contents of your backup (path/to/backupdir) to your <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracEnvironment\">project environment</a> directory and restart the service.\n</p>\n<blockquote>\n<p>\n<em>Note: Automatic backup of environments that don't use SQLite as database backend is not supported at this time. As a workaround, we recommend that you stop the server, copy the environment directory, and make a backup of the database using whatever mechanism is provided by the database system.</em>\n</p>\n</blockquote>\n<hr />\n<p>\nSee also: <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracAdmin\">TracAdmin</a>, <a class=\"wiki\"
