class=\"uidshort\">SpectrumKernel</strong>\n</pre>\n\n<hr />\n<p>Simple kernel to calculate similarity between pairs of strings. \n  Similarity is based on the number of &quot;k-mers&quot; in common between\n  the two strings.  That is, all substrings of length k.</p>\n  <p>Conceptually, a feature vector of all possible k-mers is created for \n  each string and has counts assigned to the elements for each respective \n  k-mer that exists in the string.  The dot-product between these two \n  vectors is then taken as the similarity score.</p>\n  This is a very large vector of length (n^k) where n is the number of \n  letters in the &quot;alphabet&quot; of the string.  That is, the number \n  of possible distinct characters the string can contain.  This is a sparse\n  vector, mostly 0's, thus actual such arrays are not used to represent \n  these arrays.  Instead, a &quot;feature dictionary&quot; containing only \n  found k-mers and their counts is created.<br /><br />\n\n<!-- ==================== INSTANCE METHODS ==================== -->\n<a
