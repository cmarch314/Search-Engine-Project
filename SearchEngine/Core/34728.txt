examined Clicking on the diff icon after revision number will display the changes made to the files modified in that revision Clicking on the Age of the file will take you to that changeset in the class=\"False\"><a class=\"False\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracNotification\">Notification</a></li></ul></div><p>\n</p>\n<p>\nThe Trac repository browser can be used to browse specific revisions of directories \nand files stored in the repositories associated with the Trac environment.\n</p>\n<p>\n<em>(since 0.12)</em>: \nAt the top-level of the repository browser is the <strong>Repository Index</strong>, \nlisting all the configured repositories. \nEach repository has a name which is used as a path prefix in a \n\"virtual\" file hierarchy encompassing all the available repositories.\nIf a default repository has been configured, its top-level files and directories \nare also listed, in a <strong>Default Repository</strong> section placed before the \nrepository index. If the default repository is the only repository associated \nwith the Trac environment the <strong>Repository Index</strong> will be omitted <sup><a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracBrowser#note-multirepos\">(1)</a></sup>.\n</p>\n<p>\nDirectory entries are displayed in a list with sortable columns. The list \nentries can be sorted by <em>Name</em>, <em>Size</em>, <em>Age</em> or <em>Author</em> by clicking on the column\nheaders. The sort order can be reversed by clicking on a given column\nheader again.\n</p>\n<p>\nThe browser can be used to navigate through the directory structure \nby clicking on the directory names. \nClicking on a file name will show the contents of the file. \nClicking on the revision number of a file or directory will take \nyou to the <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracRevisionLog\">TracRevisionLog</a> for that file.\nNote that there's also a <em>Revision Log</em> navigation link that will do the \nsame for the path currently being examined.\nClicking on the <em>diff</em> icon after revision number will display the changes made \nto the files modified in that revision.\nClicking on the <em>Age</em> of the file - will take you to that changeset in the timeline.\n</p>\n<p>\nIt's also possible to browse directories or files as they were in history,\nat any given repository revision. The default behavior is to display the\nlatest revision but another revision number can easily be selected using\nthe <em>View revision</em> input field at the top of the page.\n</p>\n<p>\nThe color bar next to the <em>Age</em> column gives a visual indication of the age\nof the last change to a file or directory, following the convention that\n<strong><span style=\"color:#88f\">blue</span></strong> is oldest and <strong><span style=\"color:#f88\">red</span></strong>\nis newest, but this can be <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracIni#browser-section\">configured</a>.\n</p>\n<p>\nAt the top of the browser page, there's a <em>Visit</em> drop-down menu which you can use \nto select some interesting places in the repository, for example branches or tags. \nThis is sometimes referred to as the <em>browser quickjump</em> facility.\nThe precise meaning and content of this menu depends on your repository backend.\nFor Subversion, this list contains by default the top-level trunk directory \nand sub-directories of the top-level branches and tags directories \n(<tt>/trunk</tt>, <tt>/branches/*</tt>, and <tt>/tags/*</tt>).  This can be <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracIni#svn-section\">configured</a> \nfor more advanced cases.\n</p>\n<p>\nIf you're using a Javascript enabled browser, you'll be able to expand and \ncollapse directories in-place by clicking on the arrow head at the right side of a \ndirectory. Alternatively, the <a class=\"ext-link\" href=\"http://trac.edgewall.org/intertrac/TracKeys\" title=\"TracKeys in Trac project trac\"><span class=\"icon\">\u200b</span>keyboard</a> can also be used for this: \n</p>\n<ul><li>use <tt>'j'</tt> and <tt>'k'</tt> to select the next or previous entry, starting with the first\n</li><li><tt>'o'</tt> (open) to toggle between expanded and collapsed state of the selected \ndirectory or for visiting the selected file \n</li><li><tt>'v'</tt> (view, visit) and <tt>'&lt;Enter&gt;'</tt>, same as above\n</li><li><tt>'r'</tt> can be used to force the reload of an already expanded directory\n</li><li><tt>'A'</tt> can be used to directly visit a file in annotate (blame) mode\n</li><li><tt>'L'</tt> to view the log for the selected entry\n</li></ul><p>\nIf no row has been selected using <tt>'j'</tt> or <tt>'k'</tt> these keys will operate on the entry under the mouse .\n</p>\n<p>\nFor the Subversion backend, some advanced additional features are available:\n</p>\n<ul><li>The <tt>svn:needs-lock</tt> property will be displayed\n</li><li>Support for the <tt>svn:mergeinfo</tt> property showing the merged and eligible information\n</li><li>Support for browsing the <tt>svn:externals</tt> property \n(which can be <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracIni#svn:externals-section\">configured</a>)\n</li><li>The <tt>svn:mime-type</tt> property is used to select the syntax highlighter for rendering \nthe file. For example, setting <tt>svn:mime-type</tt> to <tt>text/html</tt> will ensure the file is \nhighlighted as HTML, regardless of the file extension. It also allows selecting the character \nencoding used in the file content. For example, if the file content is encoded in UTF-8, \nset <tt>svn:mime-type</tt> to <tt>text/html;charset=utf-8</tt>. The <tt>charset=</tt> specification overrides the \ndefault encoding defined in the <tt>default_charset</tt> option of the <tt>[trac]</tt> section \nof <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracIni#trac-section\">trac.ini</a>.\n</li></ul><hr />\n<div style=\"font-size:85%\" class=\"wikipage\"><p>\n<span class=\"wikianchor\"
