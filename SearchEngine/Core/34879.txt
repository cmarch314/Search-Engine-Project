searchable\">\n        \n          <div id=\"wikipage\" class=\"trac-content\"><h1 id=\"InstallingTracasCGI\">Installing Trac as CGI</h1>\n<div class=\"important\"><blockquote>\n<p>\n<em>Please note that using Trac via CGI is the slowest deployment method available. It is slower than <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracModPython\">mod_python</a>, <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracFastCgi\">FastCGI</a> and even <a class=\"ext-link\" href=\"http://trac.edgewall.org/intertrac/TracOnWindowsIisAjp\" title=\"TracOnWindowsIisAjp in Trac project trac\"><span class=\"icon\">\u200b</span>IIS/AJP</a> on Windows.</em>\n</p>\n</blockquote>\n</div><p>\nCGI script is the entrypoint that web-server calls when a web-request to an application is made. To generate the <tt>trac.cgi</tt> script run:\n</p>\n<pre class=\"wiki\">trac-admin /path/to/env deploy /path/to/www/trac\n</pre><p>\n<tt>trac.cgi</tt> will be in the <tt>cgi-bin</tt> folder inside the given path. Make sure it is executable by your web server. This command also copies <tt>static resource</tt> files to a <tt>htdocs</tt> directory of a given destination.\n</p>\n<h2 id=\"Apacheweb-serverconfiguration\">Apache web-server configuration</h2>\n<p>\nIn <a class=\"ext-link\" href=\"http://httpd.apache.org/\"><span class=\"icon\">\u200b</span>Apache</a> there are two ways to run Trac as CGI:\n</p>\n<ol><li>Use a <tt>ScriptAlias</tt> directive that maps an URL to the <tt>trac.cgi</tt> script (recommended)\n</li><li>Copy the <tt>trac.cgi</tt> file into the directory for CGI executables used by your web server (commonly named <tt>cgi-bin</tt>). You can also create a symbolic link, but in that case make sure that the <tt>FollowSymLinks</tt> option is enabled for the <tt>cgi-bin</tt> directory.\n</li></ol><p>\nTo make Trac available at <tt>http://yourhost.example.org/trac</tt> add <tt>ScriptAlias</tt> directive to Apache configuration file, changing <tt>trac.cgi</tt> path to match your installation:\n</p>\n<pre class=\"wiki\">ScriptAlias /trac /path/to/www/trac/cgi-bin/trac.cgi\n</pre><blockquote>\n<p>\n<em>Note that this directive requires enabled <tt>mod_alias</tt> module.</em>\n</p>\n</blockquote>\n<p>\nIf you're using Trac with a single project you need to set its location using the <tt>TRAC_ENV</tt> environment variable:\n</p>\n<pre class=\"wiki\">&lt;Location \"/trac\"&gt;\n  SetEnv TRAC_ENV \"/path/to/projectenv\"\n&lt;/Location&gt;\n</pre><p>\nOr to use multiple projects you can specify their common parent directory using the <tt>TRAC_ENV_PARENT_DIR</tt> variable:\n</p>\n<pre class=\"wiki\">&lt;Location \"/trac\"&gt;\n  SetEnv TRAC_ENV_PARENT_DIR \"/path/to/project/parent/dir\"\n&lt;/Location&gt;\n</pre><blockquote>\n<p>\n<em>Note that the <tt>SetEnv</tt> directive requires enabled <tt>mod_env</tt> module. It is also possible to set TRAC_ENV in trac.cgi. Just add the following code between \"try:&#34; and &#34;from trac.web ...\":</em>\n</p>\n</blockquote>\n<pre class=\"wiki\">    import os\n    os.environ['TRAC_ENV'] = \"/path/to/projectenv\"\n</pre><blockquote>\n<p>\n<em> Or for TRAC_ENV_PARENT_DIR: </em>\n</p>\n</blockquote>\n<pre class=\"wiki\">    import os\n    os.environ['TRAC_ENV_PARENT_DIR'] = \"/path/to/project/parent/dir\"\n</pre><p>\nIf you are using the <a class=\"ext-link\" href=\"http://httpd.apache.org/docs/suexec.html\"><span class=\"icon\">\u200b</span>Apache suEXEC</a> feature please see <a class=\"ext-link\" href=\"http://trac.edgewall.org/wiki/ApacheSuexec\"><span class=\"icon\">\u200b</span>http://trac.edgewall.org/wiki/ApacheSuexec</a>.\n</p>\n<p>\nOn some systems, you <em>may</em> need to edit the shebang line in the <tt>trac.cgi</tt> file to point to your real Python installation path. On a Windows system you may need to configure Windows to know how to execute a .cgi file (Explorer -&gt; Tools -&gt; Folder Options -&gt; File Types -&gt; CGI).\n</p>\n<h2 id=\"MappingStaticResources\">Mapping Static Resources</h2>\n<p>\nOut of the box, Trac will pass static resources such as style sheets or images through itself. For a CGI setup this is <strong>highly undesirable</strong>, because this way CGI script is invoked for documents that could be much more efficiently served directly by web server.\n</p>\n<p>\nWeb servers such as <a class=\"ext-link\" href=\"http://httpd.apache.org/\"><span class=\"icon\">\u200b</span>Apache</a> allow you to create \u201cAliases\u201d to resources, giving them a virtual URL that doesn't necessarily reflect the layout of the servers file system. We already used this capability by defining a <tt>ScriptAlias</tt> for the CGI script. We also can map requests for static resources directly to the directory on the file system, avoiding processing these requests by CGI script.\n</p>\n<p>\nThere are two primary URL paths for static resources - <tt>/chrome/common</tt> and <tt>/chrome/site</tt>. Plugins can add their own resources usually accessible by <tt>/chrome/plugin</tt> path, so its important to override only known paths and not try to make universal <tt>/chrome</tt> alias for everything.\n</p>\n<p>\nAdd the following snippet to Apache configuration <strong>before</strong> the <tt>ScriptAlias</tt> for the CGI script, changing paths to match your deployment:\n</p>\n<pre class=\"wiki\">Alias /trac/chrome/common /path/to/trac/htdocs/common\nAlias /trac/chrome/site /path/to/trac/htdocs/site\n&lt;Directory \"/path/to/www/trac/htdocs\"&gt;\n  Order allow,deny\n  Allow from all\n&lt;/Directory&gt;\n</pre><p>\nIf using mod_python, you might want to add this too (otherwise, the alias will be ignored):\n</p>\n<pre class=\"wiki\">&lt;Location \"/trac/chrome/common/\"&gt;\n  SetHandler None\n&lt;/Location&gt;\n</pre><p>\nNote that we mapped <tt>/trac</tt> part of the URL to the <tt>trac.cgi</tt> script, and the path <tt>/chrome/common</tt> is the path you have to append to that location to intercept requests to the static resources. \n</p>\n<p>\nFor example, if Trac is mapped to <tt>/cgi-bin/trac.cgi</tt> on your server, the URL of the Alias should be <tt>/cgi-bin/trac.cgi/chrome/common</tt>.\n</p>\n<p>\nSimilarly, if you have static resources in a project's htdocs directory (which is referenced by /chrome/site URL in themes), you can configure Apache to serve those resources (again, put this <strong>before</strong> the <tt>ScriptAlias</tt> for the CGI script, and adjust names and locations to match your installation):\n</p>\n<pre class=\"wiki\">Alias /trac/chrome/site /path/to/projectenv/htdocs\n&lt;Directory \"/path/to/projectenv/htdocs\"&gt;\n  Order allow,deny\n  Allow from all\n&lt;/Directory&gt;\n</pre><p>\nAlternatively to hacking <tt>/trac/chrome/site</tt>, you can directly specify path to static resources using <tt>htdocs_location</tt> configuration option in <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracIni\">trac.ini</a>:\n</p>\n<pre class=\"wiki\">[trac]\nhtdocs_location = http://yourhost.example.org/trac-htdocs\n</pre><p>\nTrac will then use this URL when embedding static resources into HTML pages. Of course, you still need to make the Trac <tt>htdocs</tt> directory available through the web server at the specified URL, for example by copying (or linking) the directory into the document root of the web server:\n</p>\n<pre class=\"wiki\">$ ln -s /path/to/www/trac/htdocs /var/www/yourhost.example.org/trac-htdocs\n</pre><p>\nNote that in order to get this <tt>htdocs</tt> directory, you need first to extract the relevant Trac resources using the <tt>deploy</tt> command of <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracAdmin\">TracAdmin</a>:\n<pre class=\"wiki\">deploy &lt;directory&gt;\n\n    Extract static resources from Trac and all plugins\n\n</pre>\n</p>\n<h2 id=\"AddingAuthentication\">Adding Authentication</h2>\n<p>\nThe simplest way to enable authentication with Apache is to create a password file. Use the <tt>htpasswd</tt> program to create the password file:\n</p>\n<pre class=\"wiki\">$ htpasswd -c /somewhere/trac.htpasswd admin\nNew password: &lt;type password&gt;\nRe-type new password: &lt;type password again&gt;\nAdding password for user admin\n</pre><p>\nAfter the first user, you dont need the \"-c\" option anymore:\n</p>\n<pre class=\"wiki\">$ htpasswd /somewhere/trac.htpasswd john\nNew password: &lt;type password&gt;\nRe-type new password: &lt;type password again&gt;\nAdding password for user john\n</pre><blockquote>\n<p>\n<em>See the man page for <tt>htpasswd</tt> for full documentation.</em>\n</p>\n</blockquote>\n<p>\nAfter you've created the users, you can set their permissions using <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracPermissions\">TracPermissions</a>.\n</p>\n<p>\nNow, you'll need to enable authentication against the password file in the Apache configuration:\n</p>\n<pre class=\"wiki\">&lt;Location \"/trac/login\"&gt;\n  AuthType Basic\n  AuthName \"Trac\"\n  AuthUserFile /somewhere/trac.htpasswd\n  Require valid-user\n&lt;/Location&gt;\n</pre><p>\nIf you're hosting multiple projects you can use the same password file for all of them:\n</p>\n<pre class=\"wiki\">&lt;LocationMatch \"/trac/[^/]+/login\"&gt;\n  AuthType Basic\n  AuthName \"Trac\"\n  AuthUserFile /somewhere/trac.htpasswd\n  Require valid-user\n&lt;/LocationMatch&gt;\n</pre><p>\nFor better security, it is recommended that you either enable SSL or at least use the \u201cdigest\u201d authentication scheme instead of \u201cBasic\u201d. Please read the <a class=\"ext-link\" href=\"http://httpd.apache.org/docs/2.0/\"><span class=\"icon\">\u200b</span>Apache HTTPD documentation</a> to find out more. For example, on a Debian 4.0r1 (etch) system the relevant section  in apache configuration can look like this:\n</p>\n<pre class=\"wiki\">&lt;Location \"/trac/login\"&gt;\n    LoadModule auth_digest_module /usr/lib/apache2/modules/mod_auth_digest.so\n    AuthType Digest\n    AuthName \"trac\"\n    AuthDigestDomain /trac\n    AuthUserFile /somewhere/trac.htpasswd\n    Require valid-user\n&lt;/Location&gt;\n</pre><p>\nand you'll have to create your .htpasswd file with htdigest instead of htpasswd as follows:\n</p>\n<pre class=\"wiki\"># htdigest /somewhere/trac.htpasswd trac admin\n</pre><p>\nwhere the \"trac\" parameter above is the same as AuthName above  (\"Realm\" in apache-docs). \n</p>\n<hr />\n<p>\nSee also:  <a class=\"wiki\"
