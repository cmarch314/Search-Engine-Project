HREF=\"Class_Kernel_archetype.html#Cross_link_anchor_25\">CGAL::Kernel_archetype</A></I> is a concept archetype (minimal model) for the CGAL kernel concept.\nIt provides all functionality required by the CGAL kernel concept, but nothing more.\nIt can be used for testing successful compilation of packages of the basic library with a minimal\nmodel. Deprecated kernel functionality is not supported. All geometrical types (like the 2d/3d point or segment types)\nof <I><A HREF=\"Class_Kernel_archetype.html#Cross_link_anchor_25\">CGAL::Kernel_archetype</A></I> have\ncopy constructors, default constructors and an assignment operator, and nothing else.\nComparison operators are by default not supported, but can be switched on by defining the macro\n<I>CGAL_CONCEPT_ARCHETYPE_ALLOW_COMPARISONS</I>.\n<P>\n\nThe geometrical types of the concept archetype encapsulate no data members, so runtime checks with the archetype\nare not very useful (<I><A HREF=\"Class_Kernel_archetype.html#Cross_link_anchor_25\">CGAL::Kernel_archetype</A></I> is only meant for compilation checks with a minimal model in the testsuites\nof CGAL packages).\n<P>\n\nThe package supports the two- and three-dimensional part of the CGAL kernel concept. The d-dimensional\npart is not supported.\n<P>\n\n<I><A HREF=\"Class_Kernel_archetype.html#Cross_link_anchor_25\">CGAL::Kernel_archetype</A></I> normally offers the full functionality (all types, functors and constructions\nof a CGAL kernel model), but it is possible to restrict the interface. This can be useful\nfor testing packages that require only a very small subset of the functionality offered\nby CGAL kernel models.\nIf you want to do this, you have to define the macro <I>CGAL_CA_LIMITED_INTERFACE</I> \n(before the inclusion of <I>CGAL/<A HREF=\"Class_Kernel_archetype.html#Cross_link_anchor_26\">Kernel_archetype</A>.h</I>) to switch on the interface limitation. \nNow you have to tell the kernel archetype the types it has to provide for the specific package.\n<P>\n\nFor every type you have to define a macro.\nThe name of the macro is <I>CGAL_CA_NAME_OF_KERNEL_TYPE</I>, where <I>NAME_OF_KERNEL_TYPE</I> is the name\nof the kernel type (written in capitals) that has to be provided by the kernel archetype for a specific package.\nIf, for example, a package only needs type definitions for <I><A HREF=\"Class_Point_2.html#Cross_link_anchor_52\">Point_2</A></I> and <I>Orientation_2</I>, you would\ndefine <I>CGAL_CA_LIMITED_INTERFACE</I>, <I>CGAL_CA_POINT_2</I> and <I>CGAL_CA_ORIENTATION_2</I> .\n<P>\n\n<H3>Is Model for the Concept</H3><I><A HREF=\"Concept_Kernel.html#Cross_link_anchor_0\">Kernel</A></I>\n<P>\n\n<I>#include &lt;<A HREF=\"../../../include/CGAL/Kernel_archetype.h\">CGAL/Kernel_archetype.h</A>&gt;</I>\n<P>\n\n<H3>Creation</H3>\n<P>\n\n<TABLE BORDER=0 CELLSPACING=2 CELLPADDING=0 WIDTH=650>\n        <TR><TD ALIGN=LEFT VALIGN=TOP WIDTH=100% NOWRAP COLSPAN=2>\n        <I><NOBR>\n    <A HREF=\"Class_Kernel_archetype.html#Cross_link_anchor_26\">Kernel_archetype</A> ka;\n        </I></NOBR>\n        </TD></TR><TR><TD WIDTH=30% NOWRAP></TD><TD ALIGN=LEFT VALIGN=TOP WIDTH=70%>\n    Default constructor.\n<P>\n\n</TD></TR>\n        </TABLE>\n<P>\n\n<H3>Types</H3>\n<P>\n\nWe provide all type definitions that must be provided by a CGAL kernel model.\nSee the CGAL kernel concept manual pages for details.\nDeprecated functionality is not supported.
