text=\"#000000\">\n<h1>Cellular Automata: Replicators</h1>\n\nA <i>replicator</i> is usually defined as a cellular automaton\npattern which can make copies of itself. This definition is\nsomewhat vague, but there are so few examples known that it would\nbe unhelpful to try to be more precise. Replicators are interesting\nnot only in themselves, but because they can be used to build many\nother types of patterns: high-period oscillators, glider guns,\npuffers and spaceships, and pseudo-random number generators. \n\n<p>Most replicators work accoding to a <i>parity rule:</i> there is\na (usually one-dimensional) grid of possible replicator positions,\nsuch that, if replicators are placed at certain grid positions,\nthey produce new replicators at the positions with an odd number of\nreplicators in neighboring cells. If the grid positions are spaced\nk units apart, and the period (number of generations between each\nreplication) is p, then the resulting pattern of oscillators\nexpands at a speed of kc/p, with a <i>sawtooth</i> growth rate in\nwhich the number of active cells repeatedly increases to linear (or\nquadratic, for two dimensional grids) then decreases to a constant.\nThe minima of the number of active cells occur at generations that\nare powers of two times the period.</p>\n\n<p>Parity-rule replicators are fairly common in rules with B1\n(birth on a single neighbor, in <a href=\"../lifelike.html\">standard\nsemitotalistic notation</a>), but those rules are not very\ninteresting because any pattern quickly grows to infinity in all\ndirections, so gliders, oscillators, and similar structures are\nimpossible. The more interesting rules are those with B0, B2 or B3, in\nwhich gliders are possible (and <a href=\"../\">many gliders are\nknown</a>).
