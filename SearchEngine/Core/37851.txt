OEErrorHandler provides the following main methods:\n\n<P>\n\n<UL>\n<LI>SetOutputStream(stream)\n<BR>\nBy default, output from an OEErrorHandler instance will go to stdout. However,\nthere are times when the user may want to use stdout instead or to log\ndirectly to a file.  To use stdout, call <code>SetOutputStream(cvar.oeout)</code>\nand for stderr use <code>SetOutputStream(cvar.oeerr)</code>.  To use a file instead,\ncreate an instance of an oeofstream and then call SetOutputStream with that.\n\n<P>\n</LI>\n<LI>Info(message)\n<BR>\nUsed for pure informational messages.\n\n<P>\n</LI>\n<LI>Fatal(message)\n<BR>\nUsed to log a fatal error and die. Program will exit after logging the\nmessage.\n\n<P>\n</LI>\n<LI>Error(message)\n<BR>\nUsed to log an error.\n\n<P>\n</LI>\n<LI>Warning(message)\n<BR>\nUsed to log a warning.\n\n<P>\n</LI>\n<LI>Usage(message)\n<BR>\nMany programs use a commandline switch like ``-h'' or ``-help'' to\nshow the user a list of commandline options. Use Usage(message) to\nsend to the output stream and the exit the program.\n</LI>\n</UL>\n\n<P>\nA new OEErrorHandler class instance can be created and then\nthe output can be set at runtime to any of the available output streams,\nincluding stdout (cvar.oeout), stderr (cvar.oeerr), a file, or a string.\nAll of this functionality is available using built-in Python methods, but\nprogrammers wanting to write code for possible future porting to C++ may want\nto use this message of printing messages to make the task of porting more\nstraightforward.\n\n<P>\nHere is an example of creating an OEErrorHandler and using it to write\nto a log file or to the terminal, depending on whether a filename is provided\non the commandline.\n\n<P>\n<dl><dd><pre class=\"verbatim\">#!/usr/bin/env python\nimport os, sys\nfrom openeye.oechem import *\n\nLog = OEErrorHandler()\n\nif len(sys.argv)==2:\n    ofs = oeofstream(sys.argv[1])\nelse:\n    ofs = cvar.oeerr\n\nLog.SetOutputStream(ofs)\n\nLog.Info(\"Here is an information message\")\nLog.Warning(\"Here is a warning\")\nLog.Fatal(\"Here is a fatal message\")\n\n# program dies after Fatal above so this doesn't get called\nLog.Info(\"Shouldn't see this one\")\n</pre></dl>\n\n<P>\n\n<DIV CLASS=\"navigation\">\n<p><hr>\n<table align=\"center\" border=\"0\" width=\"100%\" cellpadding=\"0\" cellspacing=\"0\">\n<tr>\n<td rowspan=2><A HREF=\"node173.html\"><img src=\"../icons/previous.gif\"\n  border=\"0\" alt=\"Previous Page\"></A></td>\n<td rowspan=2><A HREF=\"node173.html\"><img src=\"../icons/up.gif\"\n  border=\"0\" alt=\"Up One Level\"></A></td>\n<td rowspan=2><A HREF=\"node175.html\"><img src=\"../icons/next.gif\"\n  border=\"0\" alt=\"Next Page\"></A></td><td width=\"100%\" rowspan=\"2\"> </td>\n<td valign=\"center\" align=\"right\" height=\"56\" background=\"../icons/title_bkgd.gif\" colspan=4 class=\"navigation\">OEChem -
