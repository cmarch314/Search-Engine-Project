was actually an answer that I wrote in my Software Engineering Final. The question had asked us to describe the state of the software engineering practice today (with respect to the time period of the post\u2019s date).</h5>\n\n<p>Software Practices are highly fragmented in their standards owning to the diversity of software solutions which meet a very large set of user requirements. For instance one can observe the fragmentation in the standards in the SDLC models being adopted, which basically involves a mix and match of the standard models (waterfall\u2026Agile). The same can be observed in the diversity of the implementation frameworks: .NET, Java, Rails. Such a fragmentation is never conducive for collaborations between various developing organizations. This leads to the creation of middle-ware overheads which connects collaborating entities coming from varying developer cultures. Overall this leads towards inefficient software practices which result in substandard software solutions.</p>\n \n</div>\n\n<div style=\"margin-bottom: 10px\">\n\t<br>\n\t<b class=\"label\">Tags:</b>\n\t\n\t\t<span class=\"tags\">software-engineering</span>\n\t\t\n\t\n</div>\n</div>\n\n<div class=\"plain stretch\">\n  <h2>Related Posts</h2>\n  <ul class=\"posts\">\n    \n      <li><span>11 Jan 2015</span> &raquo;
