Bibliography</A>\n<br><hr>\n</DIV>\n<!--End of Navigation Panel-->\n\n<H2><A NAME=\"SECTION0032190000000000000000\">\n30.1.9 Installing Python from Source</A>\n</H2>\n\n<P>\nIt is very likely that if you have a Unix system (other than Linux),\nPython will not be installed. However, downloading, building and\ninstalling Python is very simple and works on every platform on which\nOEChem is available. Go to http://www.python.org/download and get the\nlatest version of Python 2.3 (Python 2.3.4, as of this\nwriting). Following that link will bring you to a v2.3.4 specific\ndownload page so you can download Python-2.3.4.tgz (or similar for a\ndifferent version number).  Save this file in an empty directory and\nthen follow the simplified directions below (or better yet, follow the\nINSTALL directions found in the .tgz file)\n\n<P>\n\n<OL>\n<LI>cd to the directory containing the Python-2.3.4.tgz file.\n</LI>\n<LI>gunzip Python-2.3.4.tgz\n</LI>\n<LI>tar -xvf Python-2.3.4.tar\n</LI>\n<LI>cd Python-2.3.4\n</LI>\n<LI>./configure  The main option to configure is a decision about the root\n    of the install tree and whether or not to use gcc as the compiler.\n\n<P>\nBy default, configure will choose /usr/local such that the application\nends up in /usr/local/bin/python and the associated library files end\nup in /usr/local/lib/python2.3. Additionally, if gcc is in your path,\nthe default configure will use gcc to build Python. It is important that\nthe C compiler used to build Python match the version of the C compiler\nused to build Python-OEChem, so see the specific section below for\nyour platform.\n\n<P>\nIf you are happy with the default then just run:\n\n<P>\n<dl><dd><pre class=\"verbatim\">./configure\n</pre></dl>\n\n<P>\nIf you have an alternate place (such as /apps or /sw) the run configure as:\n\n<P>\n<dl><dd><pre class=\"verbatim\">./configure --prefix=/apps\n</pre></dl>\n\n<P>\nTo use the native, system compiler instead of gcc:\n\n<P>\n<dl><dd><pre class=\"verbatim\">./configure --without-gcc\n</pre></dl>\n\n<P>\nOn Linux, to use the system gcc in <code>/usr/bin</code> instead of a\nnewer version installed in <code>/usr/local/bin</code>:\n\n<P>\n<dl><dd><pre class=\"verbatim\">export CC=/usr/bin/gcc\n./configure\n</pre></dl>\n\n<P>\n</LI>\n<LI>run ``make''\n</LI>\n<LI>As root, run ``make install''\n</LI>\n<LI>As a normal user, start a new shell and type:\n\n<P>\n<dl><dd><pre class=\"verbatim\">which python\n</pre></dl>\n\n<P>\nand make sure it points to /usr/local/bin/python if you ran configure without a\nprefix option or to /apps/bin/python, /sw/bin/python, etc. depending on the\nactual prefix you fed configure in step 5.\n\n<P>\n</LI>\n</OL>\n\n<P>\n\n<DIV CLASS=\"navigation\">\n<p><hr>\n<table align=\"center\" border=\"0\" width=\"100%\" cellpadding=\"0\" cellspacing=\"0\">\n<tr>\n<td rowspan=2><A HREF=\"node264.html\"><img src=\"../icons/previous.gif\"\n  border=\"0\" alt=\"Previous Page\"></A></td>\n<td rowspan=2><A HREF=\"node256.html\"><img src=\"../icons/up.gif\"\n  border=\"0\" alt=\"Up One Level\"></A></td>\n<td rowspan=2><A HREF=\"node266.html\"><img src=\"../icons/next.gif\"\n  border=\"0\" alt=\"Next Page\"></A></td><td width=\"100%\" rowspan=\"2\"> </td>\n<td valign=\"center\" align=\"right\" height=\"56\" background=\"../icons/title_bkgd.gif\" colspan=4 class=\"navigation\">OEChem -
