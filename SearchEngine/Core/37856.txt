For these instances,\noemolstreams provide the <code>SetFlavor</code> and <code>GetFlavor</code>\nmethods.\n\n<P>\nThe <code>SetFlavor</code> function takes two integer arguments, the first\nis the format for which the flavor is being specified and the second is the\nflavor itself.  The formats are specified as discussed above for\n<code>SetFormat</code>.  The input flavors are specified in the OEChem namespace\nOEIFlavor and the output flavors are specified int the OEChem namespace\nOEOFlavor.  Unlike the formats, the flavors are a bitmask and may be or'ed\ntogether.  Under the OEIFlavor and OEOFlavor namespaces, there is a\nnamespace for each format as well as a generic namespace.  The generic\nnamespace is used to control aromaticity perception and other properties\ncommon to all of formats.  To completely specify a flavor, one would\ntypically binary-OR a generic flag and a format specific flag and pass the\nresultant value to <code>SetFlavor</code>.\n\n<P>\nThe default behavior for the PDB reader is that TER specifies the\ntermination of a disconnected fragment within the same molecule while END\nspecified the termination of a connection table (see the API manual for\ndetails).  However, some users may want to have the reader split PDB input\nfiles into different molecules every time a TER appears.\n\n<P>\nThe following code is an example of changing the PDB reader flavor.\n\n<P>\n<dl><dd><pre class=\"verbatim\">#!/usr/bin/env python\n#ch4-9.py\n\nfrom openeye.oechem import *\nimport sys\n\nifs = oemolistream('input.pdb')\nofs = oemolostream('output.mol2')\n\nflavor = OEIFlavor_Generic_Default|OEIFlavor_PDB_Default|OEIFlavor_PDB_TER\nifs.SetFlavor(OEFormat_PDB, flavor)\n\nfor mol in ifs.GetOEMols():\n    OEWriteMolecule(ofs, mol)\n</pre>\n<div class=\"verbatiminput-footer\">\n<a href=\"ch4-9.txt\" type=\"text/plain\">Download as text.</a>\n</div>\n</dd></dl>\n\n<P>\nSimilar low-level control can be exerted over both input and output stream\nreaders using the powerful <code>SetFlavor</code> command.
