All possible transformations are applied to the initial set\nof atoms and bonds of an input molecule.  For example, a reaction that\naffects a particular type of functional group will be automatically be\napplied twice a to bi-functional molecule.  The number of\ntransformations applied by the <code>OEUniMolecularRxn</code> class is\nlimited in order to prevent infinite loops.  Consider a hypothetical\nreaction that methylates a methyl group.  If a methyl group added in a\nreaction were allowed to react again, the methyl groups of a molecule\nwould be methylated <i>ad infinitum</i>.  The first protection against\ninfinite loops provided by <code>OEUniMolecularRxn</code> is that only\noriginal atoms and bonds of the input molecule are allowed to react.\nAtoms and bonds created by a reaction are excluded from involvement in\nfurther reactions.  A more subtle source of potential infinite loops\nare reactions where products atoms still match the reactant pattern\nafter they have been involved in a chemical transformation.  The\n<code>OEUniMolecularRxn</code> class allows a set of atom that match a\nreactant pattern to react only a single time.\n\n<P>\nThe following code demonstrates the use of the\n<code>OEUniMolecularRxn</code> class.  The <code>OEUniMolecularRxn</code> in this\ncase is initialized using a SMIRKS pattern.  The example reaction\nprotonates and charges an amine nitrogen.  When the <code>OEUniMolecularRxn</code>\nclass is applied to 1,2-ethanediamine both nitrogens are charged and\nprotonated to yield 1,2-ethanediaminium.  The example reaction was\nintentionally written to demonstrate the protection mechanisms in place\nto prevent underspecified reactions from causing infinite loops.  The\nproduct 1,2-ethanediaminium still matches the reactant pattern,\nhowever, subsequent reactions terminate when no unreacted atoms are\nidentified.\n\n<P>\n<dl><dd><pre class=\"verbatim\">#include \"oechem.h\"\n#include &lt;iostream&gt;\n\nusing namespace OEChem;\nusing namespace std;\n\nint main()\n{\n  OEUniMolecularRxn umr(\"[N:1]&gt;&#62;[Nh3+:1]\");\n\n  OEGraphMol mol;\n  OEParseSmiles(mol,\"NCCN\");\n  umr(mol);\n\n  std::string smi;\n  OECreateSmiString(smi,mol);\n\n  cout &lt;&#60; \"smiles = \" &lt;&#60; smi &lt;&#60; endl;\n\n  return 0;\n}\n</pre>\n<div class=\"verbatiminput-footer\">\n<a href=\"ch30-1.txt\" type=\"text/plain\">Download as text.</a>\n</div>\n</dd></dl>\n\n<P>\n\n<DIV CLASS=\"navigation\">\n<p><hr>\n<table align=\"center\" border=\"0\" width=\"100%\" cellpadding=\"0\" cellspacing=\"0\">\n<tr>\n<td rowspan=2><A HREF=\"node247.html\"><img src=\"../icons/previous.gif\"\n  border=\"0\" alt=\"Previous Page\"></A></td>\n<td rowspan=2><A HREF=\"node247.html\"><img src=\"../icons/up.gif\"\n  border=\"0\" alt=\"Up One Level\"></A></td>\n<td rowspan=2><A HREF=\"node249.html\"><img src=\"../icons/next.gif\"\n  border=\"0\" alt=\"Next Page\"></A></td><td width=\"100%\" rowspan=\"2\"> </td>\n<td valign=\"center\" align=\"right\" height=\"56\" background=\"../icons/title_bkgd.gif\" colspan=4 class=\"navigation\">OEChem - C++ Theory Manual\n<BR>\nVersion 1.3.1</td></tr>\n<tr><td><img src=\"../icons/spacer_1.gif\"\n  border=\"0\" alt=\"\"></td>\n<td><A href=\"contents.html\"><img src=\"../icons/contents.gif\"\n  border=\"0\" alt=\"Contents\"></A></td>\n<td><img src=\"../icons/spacer_2.gif\"\n  border=\"0\" alt=\"\"></td>\n<td><img src=\"../icons/blank_4.gif\"\n  border=\"0\" alt=\"\"></td>\n</tr></table>\n<b class=\"navlabel\">Previous:</b> <a class=\"sectref\" HREF=\"node247.html\">30.
