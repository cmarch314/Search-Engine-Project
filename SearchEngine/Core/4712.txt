for though</h3>\n<p><strong>Q.</strong> Do you think the JDK uses JUnit for testing? Remember, Junit came well after the JDK.</p>\n\n<hr />\n\n<h2 id=\"jdepend\">JDepend</h2>\n\n<h3 id=\"what-is-jdepend--what-does-jdepend-do\">What is JDepend? / What does JDepend do?</h3>\n<p>IMHO JDepend is a simple tool for measuring certain metrics about how packages in Java depend on each other.  </p>\n\n<p>According to <a href=\"http://www.clarkware.com/software/JDepend.html\">JDepend\u2019s Official Webpage</a>:</p>\n\n<blockquote>\n  <p>JDepend traverses Java class file directories and generates design quality metrics for each Java package, including:</p>\n\n  <ul>\n    <li>Number of Classes and Interfaces</li>\n    <li>Afferent Couplings (Ca) - measure of Responsibility</li>\n    <li>Efferent Couplings (Ce) - measure of Independence</li>\n    <li>Abstractness (A) - Number of Abstract Classes and Interfaces</li>\n    <li>Instability (I)</li>\n    <li>Distance from the Main Sequence (D)</li>\n    <li>Package Dependency Cycles </li>\n  </ul>\n</blockquote>\n\n<p>Further <a href=\"http://www.clarkware.com/software/JDepend.html#overview\">details on the metrics here\u2026</a></p>\n\n<h3 id=\"why-do-i-like-it\">Why do I like it?</h3>\n<p>It is a simple tool that succinctly captures how one Java package depends on another. This might be useful when building large projects or APIs to maintain \u201cappropriate\u201d (runtime) boundaries between different sections of your code base.</p>\n\n<h4 id=\"why-the-double-quotes-around-appropriate\">Why the double quotes around \u2018appropriate\u2019?</h4>\n<ul>\n  <li>\u2018Appropriate\u2019 is what you define for your project.</li>\n  <li>E.g. in a project following a client server architecture, you might want the client and the server to communicate via a middle-ware or an interface. Keeping the client code from calling the server code. OR, you very well could want the client code to call the server code directly.  </li>\n  <li>You might want to avoid both the client and the server calling each others modules or methods. OR, perhaps you want them to communicate like that.  </li>\n</ul>\n\n<h3 id=\"installation-and-learning-curve\">Installation and Learning curve</h3>\n\n<ul>\n  <li>INSTALLATION: I have tried the installation with Eclipse and it is simple.\n    <ul>\n      <li>Here is the <a href=\"http://andrei.gmxhome.de/jdepend4eclipse/index.html\">link for the same</a>.</li>\n      <li>This is the <a href=\"http://andrei.gmxhome.de/eclipse/\">link for installing the plugin in Eclipse</a></li>\n      <li>This is a good resource to learn how to <a href=\"http://agile.csc.ncsu.edu/SEMaterials/tutorials/install_plugin/index_v35.html\">install any Eclipse Plugin</a>  </li>\n    </ul>\n  </li>\n  <li>LEARNING CURVE: As long as you know what packages are and how one package uses another, the learning curve is non-existent. I have not yet tried using it in an advanced manner, like with JUnit or with Ant. There might be some learning curve there but it should be trivial.</li>\n</ul>\n \n</div>\n\n<div style=\"margin-bottom: 10px\">\n\t<br>\n\t<b class=\"label\">Tags:</b>\n\t\n\t\t<span class=\"tags\">testing</span>\n\t\t\n\t\t\t<span> </span>\n\t\t\n\t\n\t\t<span class=\"tags\">analysis</span>\n\t\t\n\t\t\t<span> </span>\n\t\t\n\t\n\t\t<span class=\"tags\">java</span>\n\t\t\n\t\t\t<span> </span>\n\t\t\n\t\n\t\t<span class=\"tags\">junit</span>\n\t\t\n\t\n</div>\n</div>\n\n<div class=\"plain stretch\">\n  <h2>Related Posts</h2>\n  <ul class=\"posts\">\n    \n      <li><span>11 Jan 2015</span> &raquo;
