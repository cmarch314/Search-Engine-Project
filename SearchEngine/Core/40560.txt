tags have no closing tag and should be interpreted as being\n  closed as soon as they are encountered.\n\n* The text inside some tags (ie. 'script') may contain tags which\n  are not really part of the document and which should be parsed\n  as text, not tags. If you want to parse the text as tags, you can\n  always fetch it and parse it explicitly.\n\n* Tag nesting rules:\n\n  Most tags can't be nested at all. For instance, the occurance of\n  a &lt;p&gt; tag should implicitly close the previous &lt;p&gt; tag.\n\n   &lt;p&gt;Para1&lt;p&gt;Para2\n    should be transformed into:\n   &lt;p&gt;Para1&lt;/p&gt;&lt;p&gt;Para2\n\n  Some tags can be nested arbitrarily. For instance, the occurance\n  of a &lt;blockquote&gt; tag should _not_ implicitly close the previous\n  &lt;blockquote&gt; tag.\n\n   Alice said: &lt;blockquote&gt;Bob said: &lt;blockquote&gt;Blah\n    should NOT be transformed into:\n   Alice said: &lt;blockquote&gt;Bob said: &lt;/blockquote&gt;&lt;blockquote&gt;Blah\n\n  Some tags can be nested, but the nesting is reset by the\n  interposition of other tags. For instance, a &lt;tr&gt; tag should\n  implicitly close the previous &lt;tr&gt; tag within the same &lt;table&gt;,\n  but not close a &lt;tr&gt; tag in another table.\n\n   &lt;table&gt;&lt;tr&gt;Blah&lt;tr&gt;Blah\n    should be transformed into:\n   &lt;table&gt;&lt;tr&gt;Blah&lt;/tr&gt;&lt;tr&gt;Blah\n    but,\n   &lt;tr&gt;Blah&lt;table&gt;&lt;tr&gt;Blah\n    should NOT be transformed into\n   &lt;tr&gt;Blah&lt;table&gt;&lt;/tr&gt;&lt;tr&gt;Blah\n\nDiffering assumptions about tag nesting rules are a major source\nof problems with the BeautifulSoup class. If BeautifulSoup is not\ntreating as nestable a tag your page author treats as nestable,\ntry ICantBelieveItsBeautifulSoup, MinimalSoup, or\nBeautifulStoneSoup before writing your own
