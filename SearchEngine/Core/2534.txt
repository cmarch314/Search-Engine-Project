It can\n\t  map short and long reads, and supports Hamming distance\n\t  (only substitutions) and edit distance\n\t  (substitutions/insertions/deletions). Hobbes2 accepts both\n\t  single-end and paired-end reads for alignment, and can run\n\t  on multiple CPU cores using multithreading. It supports\n\t  three input formats (Fastq, Fasta, and text files) and the\n      <a href=\"#sam-output-format\">SAM output format</a>. produces results in the SAM format. It outputs one mapping per\n        line. A single read may appear at multiple lines, where the primary\n        mapping is placed first. Reads that are unmapped are not printed. Each\n        line has the following tab separated fields:</p>\n       <p>\n\n       <small>\n       <ol>\n       <li>Name of the read mapped.</li><br>\n       <li>SAM bitwise FLAG.</li><br>\n       <li>RNAME: Reference sequence name of the mapping.\n         If @SQ header lines are present, RNAME must be present in one of the SQ-SN tag. </li><br>\n       <li>POS: 1-based leftmost mapping POSition of the first matching base.\n         The first base in a reference sequence has coordinate 1.</li><br>\n       <li>MAPQ: Mapping Quality. A value 255 indicates that the mapping quality is not available.\n         Since we don't support this yet, it's set to 255.</li><br>\n       <li>CIGAR: CIGAR string. </li><br>\n       <li>RNEXT: Reference sequence name of the NEXT fragment in the template. Currently unavailable and hence set to`*'.</li><br>\n       <li>PNEXT: Position of the NEXT fragment in the template.\n         In single-end alignment, it's set to 0; in paired-end alignment, it's the positon of it's mate pair.</li><br>\n       <li>TLEN: Signed insert size, it is set as 0 for single-end reads or when the information is unavailable.</li><br>\n       <li>SEQ: Read sequence.
