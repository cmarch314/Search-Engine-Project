class <I><A HREF=\"Class_Segment_Voronoi_diagram_filtered_traits_2.html#Cross_link_anchor_1058\">Segment_Voronoi_diagram_filtered_traits_2</A>&lt;CK,CM,EK,EM,FK,FM&gt;</I> provides a model for the\n<I><A HREF=\"Concept_SegmentVoronoiDiagramTraits_2.html#Cross_link_anchor_1052\">SegmentVoronoiDiagramTraits_2</A></I> concept.\n<P>\n\nThe class <I><A HREF=\"Class_Segment_Voronoi_diagram_filtered_traits_2.html#Cross_link_anchor_1058\">Segment_Voronoi_diagram_filtered_traits_2</A>&lt;CK,CM,EK,EM,FK,FM&gt;</I> uses the filtering technique [<A HREF=\"../biblio.html#Biblio_cgal:bbp-iayed-01\">BBP01</A>]\nto achieve traits for the <I><A HREF=\"Class_Segment_Voronoi_diagram_2.html#Cross_link_anchor_1041\">Segment_Voronoi_diagram_2</A>&lt;Gt,DS&gt;</I>\nclass with efficient and exact predicates given an exact\nkernel <I>EK</I> and a filtering kernel <I>FK</I>. The geometric\nconstructions associated provided by this class are equivalent\nto those provided by the traits class\n<I><A HREF=\"Class_Segment_Voronoi_diagram_traits_2.html#Cross_link_anchor_1054\">Segment_Voronoi_diagram_traits_2</A>&lt;CK,CM&gt;</I>, which means that\nthey may be inexact depending on the choice of the <I>CK</I> kernel.\n<P>\n\nThis class has six template parameters. The first, third and fifth\ntemplate parameters must be a models of the <I><A HREF=\"../Kernel_23_ref/Concept_Kernel.html#Cross_link_anchor_0\">Kernel</A></I> concept. The\nparameter <I>CK</I> is the construction kernel and it is the kernel\nthat will be used for constructions. The parameter <I>FK</I> is the\nfiltering kernel; this kernel will be used for performing the\narithmetic filtering for the predicates involved in the computation of\nthe segment Voronoi diagram. Finally, the parameter <I>EK</I> is the\nexact kernel; this kernel will be used for computing the predicates if\nthe filtering kernel fails to produce an answer.\n<P>\n\nThe second, fourth and sixth template parameters correspond to how\npredicates are evaluated. There are two predefined possible values for\nthese parameters, namely <I><A HREF=\"../NumberTypeSupport_ref/Class_Sqrt_field_tag.html#Cross_link_anchor_1415\">CGAL::Sqrt_field_tag</A></I> and\n<I><A HREF=\"../NumberTypeSupport_ref/Class_Field_tag.html#Cross_link_anchor_1349\">CGAL::Field_tag</A></I>. The first one must be used when the number type\nused in the representation supports the exact evaluation of signs of\nexpressions involving all four basic operations and <A HREF=\"../NumberTypeSupport_ref/Function_square.html#Cross_link_anchor_1418\">square</A> roots,\nwhereas the second one requires that only field operations are\nexact. Finally, in order to get exact constructions <I>CM</I>\nmust be set to <I><A HREF=\"../NumberTypeSupport_ref/Class_Sqrt_field_tag.html#Cross_link_anchor_1415\">CGAL::Sqrt_field_tag</A></I> and the number type in\n<I>CK</I> must support operations involing divisions and <A HREF=\"../NumberTypeSupport_ref/Function_square.html#Cross_link_anchor_1418\">square</A> roots\n(as well as the other three basic operations of course).\nThe way the predicates are evaluated is discussed in\n[<A HREF=\"../biblio.html#Biblio_b-ecvdl-96\">Bur96</A>] and [<A HREF=\"../biblio.html#Biblio_cgal:k-reisv-04\">Kar04</A>] (the geometric filtering\npart).\n<P>\n\nThe default values for the template parameters are as follows:\n<I>CM = <A HREF=\"../NumberTypeSupport_ref/Class_Sqrt_field_tag.html#Cross_link_anchor_1415\">CGAL::Sqrt_field_tag</A></I> (it is assumed that\n<I><A HREF=\"../Kernel_23_ref/Class_Cartesian.html#Cross_link_anchor_5\">CGAL::Cartesian</A>&lt;double&gt;</I> or <I><A HREF=\"../Kernel_23_ref/Class_Simple_cartesian.html#Cross_link_anchor_29\">CGAL::Simple_cartesian</A>&lt;double&gt;</I>\nwill be the entry for the template parameter <I>CK</I>), \n<I>EM = <A HREF=\"../NumberTypeSupport_ref/Class_Field_tag.html#Cross_link_anchor_1349\">CGAL::Field_tag</A></I>,\n<I>FK = <A HREF=\"../Kernel_23_ref/Class_Simple_cartesian.html#Cross_link_anchor_29\">CGAL::Simple_cartesian</A>&lt;<A HREF=\"../NumberTypeSupport_ref/Class_Interval_nt.html#Cross_link_anchor_1365\">CGAL::Interval_nt</A>&lt;false&gt; &gt;</I>,\n<I>FM = <A HREF=\"../NumberTypeSupport_ref/Class_Sqrt_field_tag.html#Cross_link_anchor_1415\">CGAL::Sqrt_field_tag</A></I>. If the G<SMALL>MP</SMALL> package is\ninstalled with C<SMALL>GAL</SMALL>, the template parameter <I>EK</I> has the default\nvalue: <I>EK = <A HREF=\"../Kernel_23_ref/Class_Simple_cartesian.html#Cross_link_anchor_29\">CGAL::Simple_cartesian</A>&lt;<A HREF=\"../NumberTypeSupport_ref/Class_Gmpq.html#Cross_link_anchor_1361\">CGAL::Gmpq</A>&gt;</I>, otherwise its\ndefault value is\n<I>EK = <A
