id=\"SyntaxColoringofSourceCode\">Syntax Coloring of Source Code</h1>\n<p>\nTrac supports language-specific syntax highlighting of source code within wiki formatted text in <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/WikiProcessors#CodeHighlightingSupport\">wiki processors</a> blocks and in the <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracBrowser\">repository browser</a>.\n</p>\n<p>\nTo do this, Trac uses external libraries with support for a great number of programming languages.\n</p>\n<p>\nCurrently Trac supports syntax coloring using one or more of the following packages:\n</p>\n<ul><li><a class=\"ext-link\" href=\"http://pygments.pocoo.org/\"><span class=\"icon\">\u200b</span>Pygments</a>, by far the preferred system, as it covers a wide range of programming languages and other structured texts and is actively supported\n</li><li><a class=\"ext-link\" href=\"http://www.codento.com/people/mtr/genscript/\"><span class=\"icon\">\u200b</span>GNU Enscript</a>, commonly available on Unix but somewhat unsupported on Windows\n</li><li><a class=\"ext-link\" href=\"http://silvercity.sourceforge.net/\"><span class=\"icon\">\u200b</span>SilverCity</a>, legacy system, some versions can be <a class=\"ext-link\" href=\"http://trac.edgewall.org/wiki/TracFaq#why-is-my-css-code-not-being-highlighted-even-though-i-have-silvercity-installed\"><span class=\"icon\">\u200b</span>problematic</a>\n</li></ul><p>\nTo activate syntax coloring, simply install either one (or more) of these packages (see <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracSyntaxColoring#ExtraSoftware\">#ExtraSoftware</a> section below).\nIf none of these packages is available, Trac will display the data as plain text. \n</p>\n<h3 id=\"AboutPygments\">About Pygments</h3>\n<p>\nStarting with trac 0.11 <a class=\"ext-link\" href=\"http://pygments.org/\"><span class=\"icon\">\u200b</span>pygments</a> will be the new default highlighter. It's a highlighting library implemented in pure python, very fast, easy to extend and <a class=\"ext-link\" href=\"http://pygments.org/docs/\"><span class=\"icon\">\u200b</span>well documented</a>.\n</p>\n<p>\nThe Pygments default style can specified in the <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracIni#mimeviewer-section\">mime-viewer</a> section of trac.ini. The default style can be overridden by setting a Style preference on the <a href=\"/LUCICodeRepository/nomaticIM/prefs/pygments\">preferences page</a>. \n</p>\n<p>\nIt's very likely that the list below is outdated because the list of supported pygments lexers is growing weekly. Just have a look at the page of <a class=\"ext-link\" href=\"http://pygments.org/docs/lexers/\"><span class=\"icon\">\u200b</span>supported lexers</a> on the pygments webpage.\n</p>\n<h2
