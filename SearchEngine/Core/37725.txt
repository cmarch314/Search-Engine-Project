This sets\nthe partial charge property of each atom, using the\n<code>OEAtomBase::SetPartialCharge</code> method.  The algorithm itself\nreproduces the partial charges as calculated by Tripos Inc's Sybyl\nsoftware, with default parameter settings, which is the de facto\nreference implementation for Gasteiger charges.\n\n<P>\nThe Gasteiger partial charge algorithm currently assumes that all\nhydrogen atoms are represented explicitly, for example by calling\n<code>OEAddExplicitHydrogens</code>.\n\n<P>\nThe current version of <code>OEGasteigerPartialCharges</code> should return\nthe same results independent of the currently assigned aromaticity\nmodels and the values of each atom's \"integer atom type\" property.\nEarly versions of OEChem allowed customization of the Gasteiger\ncharge calculation by explicitly assigning the Tripos atoms types\nof each atom of the molecule.  Unfortunately, this required assigning\nTripos' aromaticity and assigning Tripos atom types before each\ncall to <code>OEGasteigerPartialCharges</code>.  The more recent behavior\nis less error-prone (as it's no longer possible to forget to prepare\na molecule) and greatly simplifies common usage.\n\n<P>\nThe first stage of the Marsilli-Gasteiger ``Partial Equalization of\nOrbital Electronegativities (PEOE)'' calculation is assignment of\nseed charges to each atoms.  Typically for neutral atoms the partial\ncharges is seeded zero, but for example, each oxygen in a carboxylate\nis assigned -0.5, and the net formal charge on conjugated rings is\nshared equally amongst the atoms of the ring system.  These seed\ncharges may also be useful in some applications, and can be assigned\nusing the <code>OEGasteigerInitialCharges</code> method.\n\n<P>\n\n<DIV CLASS=\"navigation\">\n<p><hr>\n<table align=\"center\" border=\"0\" width=\"100%\" cellpadding=\"0\" cellspacing=\"0\">\n<tr>\n<td rowspan=2><A HREF=\"node146.html\"><img src=\"../icons/previous.gif\"\n  border=\"0\" alt=\"Previous Page\"></A></td>\n<td rowspan=2><A HREF=\"node143.html\"><img src=\"../icons/up.gif\"\n  border=\"0\" alt=\"Up One Level\"></A></td>\n<td rowspan=2><A HREF=\"node148.html\"><img src=\"../icons/next.gif\"\n  border=\"0\" alt=\"Next Page\"></A></td><td width=\"100%\" rowspan=\"2\"> </td>\n<td valign=\"center\" align=\"right\" height=\"56\" background=\"../icons/title_bkgd.gif\" colspan=4 class=\"navigation\">OEChem - Python Theory Manual\n<BR>\nVersion 1.3.1</td></tr>\n<tr><td><img src=\"../icons/spacer_1.gif\"\n  border=\"0\" alt=\"\"></td>\n<td><A href=\"contents.html\"><img src=\"../icons/contents.gif\"\n  border=\"0\" alt=\"Contents\"></A></td>\n<td><img src=\"../icons/spacer_2.gif\"\n  border=\"0\" alt=\"\"></td>\n<td><A href=\"genindex.html\"><img src=\"../icons/index.gif\"\n  border=\"0\" alt=\"Index\"></A></td>\n</tr></table>\n<b class=\"navlabel\">Previous:</b> <a class=\"sectref\" HREF=\"node146.html\">16.3 Determining Net Charge</A>\n<b class=\"navlabel\">Up:</b> <a class=\"sectref\" HREF=\"node143.html\">16.
