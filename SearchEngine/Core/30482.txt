These are goals\r\n<A NAME=588>&#160;</A>\r\npreceded by either the symbol ``?-'' or the symbol ``:-''. As soon as a query\r\nor command is encountered in the compilation of a file, the <A NAME=tex2html190 HREF=\"http://www.ecrc.de/eclipse/eclipse.html\"><b> ECLiPSe</b></A>\r\n  system will\r\ntry to satisfy it. In this way, in particular, a file can contain a directive\r\nto the system to compile another file, and so large programs can be split\r\nbetween files.\r\n<A NAME=589>&#160;</A>\r\nWhen this happens, <A NAME=tex2html192 HREF=\"http://www.ecrc.de/eclipse/eclipse.html\"><b> ECLiPSe</b></A>\r\n  interprets the pathnames of the nested\r\ncompiled files relative to the directory of the parent compiled file;\r\nif e.g. the user calls\r\n<blockquote> <PRE>[eclipse 1]: compile('src/pl/prog').\r\n</PRE>\r\n</blockquote>\r\nand the file src/pl/prog.pl contains a query\r\n<blockquote> <PRE>:- [part1, part2].\r\n</PRE>\r\n</blockquote>\r\nthen the system searches for the files <tt> part1.pl</tt> and <tt> part2.pl</tt> in the\r\ndirectory <tt> src/pl</tt> and not in the current directory.\r\nUsually larger Prolog programs have one main file which contains\r\nonly commands to compile all the subfiles.\r\nIn <A NAME=tex2html193 HREF=\"http://www.ecrc.de/eclipse/eclipse.html\"><b> ECLiPSe</b></A>\r\n  it is possible to compile this main file from any directory,\r\nwhereas in other Prolog systems it might be necessary to\r\nmake the directory of the main file the current one, or to specify\r\nin the main file the full pathnames for all compiled subfiles.\r\n<P>\r\n<A NAME=597>&#160;</A>\r\nIf the <b> compile/1</b>  predicate is called and the system is unable\r\nto find or open the required file, it will issue an error:\r\n<blockquote> <PRE>[eclipse 1]: [file].\r\nFile does not exist in compile('/user/lp/eclipse/src/file')\r\nyes.\r\n[eclipse 2]:\r\n</PRE>\r\n</blockquote>\r\n<P>\r\nIf in compilation of a file <tt> file_a</tt> the compiler is directed to open\r\nor find a file <tt>
