class=\"False\"><a class=\"False\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracNotification\">Notification</a></li></ul></div><p>\n</p>\n<p>\nWiki page names commonly use the <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/CamelCase\">CamelCase</a> convention. Within wiki text, any word in <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/CamelCase\">CamelCase</a> automatically becomes a hyperlink to the wiki page with that name.\n</p>\n<p>\n<a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/CamelCase\">CamelCase</a> page names must follow these rules:\n</p>\n<ol><li>The name must consist of <strong>alphabetic characters only</strong>. No digits, spaces, punctuation, or underscores are allowed.\n</li><li>A name must have at least two capital letters.\n</li><li>The first character must be capitalized.\n</li><li>Every capital letter must be followed by one or more lower-case letters. \n</li><li>The use of slash ( / ) is permitted in page names (possibly representing a hierarchy).\n</li></ol><p>\nIf you want to create a wiki page that doesn't follow <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/CamelCase\">CamelCase</a> rules you can use the following syntax:\n</p>\n<pre class=\"wiki\"> * [wiki:Wiki_page], [wiki:ISO9000],\n   and with a label: [wiki:ISO9000 ISO 9000 standard]\n * [wiki:\"Space Matters\"]\n   and with a label: [wiki:\"Space Matters\" all about white space]\n * or simply: [\"WikiPageName\"]s\n * even better, the new [[WikiCreole link style]]\n   and with a label: [[WikiCreole link style|WikiCreole style links]]\n</pre><p>\nThis will be rendered as:\n</p>\n<ul><li><a class=\"missing wiki\">Wiki_page?</a>, <a class=\"missing wiki\">ISO9000?</a>,\nand with a label: <a class=\"missing wiki\">ISO 9000 standard?</a>\n</li><li><a class=\"missing wiki\">Space Matters?</a> <em>(that page name embeds space characters)</em>\nand with a label: <a class=\"missing wiki\">all about white space?</a>\n</li><li>or simply: <a class=\"missing wiki\">WikiPageName?</a>s <em>(old MoinMoin's internal free links style)</em>\n</li><li>even better, the new <a class=\"missing wiki\">WikiCreole link style?</a>\nand with a label: <a class=\"missing wiki\">WikiCreole style links?</a>\n<em>(since 0.12, also now adopted by MoinMoin)</em>\n</li></ul><p>\nStarting with Trac 0.11, it's also possible to link to a specific <em>version</em> of a Wiki page, as you would do for a specific version of a file, for example: <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/WikiStart?version=1\">WikiStart@1</a>.\n</p>\n<p>\nYou can also prevent a <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/CamelCase\">CamelCase</a> name to be interpreted as a <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracLinks\">TracLinks</a>, by quoting it. See <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracLinks#EscapingLinks\">TracLinks#EscapingLinks</a>.\n</p>\n<p>\nAs visible in the example above, one can also append an anchor to a Wiki page name, in order to link to a specific section within that page. The anchor can easily be seen by hovering the mouse over a section heading, then clicking on the &para; sign that appears at its end. The anchor is usually generated automatically, but it's also possible to specify it explicitly: see <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/WikiFormatting#using-explicit-id-in-heading\">WikiFormatting#using-explicit-id-in-heading</a>.\n</p>\n<hr />\n<p>\nSee also: <a class=\"wiki\"
