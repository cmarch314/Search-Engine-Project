follows is inspired by the framework described in Toward Common Event Model for Multimedia Applications In this framework a common event model is proposed in order to allow integration and syndication of events and media from largely isolated multimedia applications and data sources making possible novel cross application and crossdata source multimedia services The Event Is a significant occurrence that takes place in point in time Is a first class entity that is independent on the media documenting it that is an event is not considered as a metadata describing a medium instead media are considered metadata documenting the course of an event Its model should provide means for global that is media independent unique identification The event model s identification system permits the unique identification of events in a uniform fashion regardless of the events sources Has 6 basic aspects spatial temporal informational experiential structural causal The aspects of an event description are shortly described below As an example a soccer match event is considered Temporal Aspect This aspect concerns the temporal awareness contained in an event description There are different ways of capturing the temporal aspect Physical time the instant or time interval when the event happens It can be measured as a global time e g date time a relative time e g minutes of game or a media related time e g frame numbers of a video documenting an event The global and relative time can be modeled by the TimeInterval e type previously described Logical time application domain s abstract temporal concept which describes the event e g the last match of the season Spatial Aspect This aspect concerns the location awareness contained in an event description There are different ways of capturing the spatial aspect Physical Location the geographical place where the event takes place A physical location is characterized by GPS coordinates city and country specifications etc For example the soccer match takes place at the Olimpic Stadium of Rome Logical Location refers to a spatial domain concept e g the match takes place in a stadium Informational Aspect This aspect concerns information about the event They can include the event type e g soccer match the event description e g the Inter Juve soccer match of the Champions League the entities agent and artifacts involved e g players and ball the role of such entities e g forward goalkeeper ball other parameters e g current score number of players etc the event mode that is how the event takes place it coulbe an online event or a physical event Experiential Aspect To augment the exploration and experience of an event a media aware component should be associated to it A set of media e g images videos audios etc should thus documents how a past event evolved Structural Aspect This aspect concerns the decomposition of high level events into low level events Sub events can be aggregated so to form more composite events e g a set of matches forms a season Knowledge on subevents can give useful hints on the composite event Causal Aspect This aspect concerns the discovering of a chain of events which led to the event in question Given an event there should be an explicit representation of such cause events chain Event Relationships The last two aspects Structural and Causal put in relation more events Here comes the issue of devising relationship between events Some types of relationship are listed below 1 Structural Connect two events by expliciting a part of relationship 2 Causal Connect two events by expliciting a sort of caused by relationship 3 Temporal Connect two events by expliciting a temporal relationship e g event A occurred during event B 4 Spatial Connect two events by expliciting a topological relation e g event A takes place in the same place of event B edit Event Metadata In view of what has been said before some metadata are proposed for the Event e type The string represents a comment Event Thing attributes period TimeInterval Temporal Aspect logicalTime String location Location Spatial Aspect abstract String Informational Aspect mode String participant &#8230;}</p>\n<p>Metadata for an event provides descriptive information that is the basis for discovery of the purpose, location, duration, and responsible agents associated with an event. Examples include an exhibition, webcast, conference, workshop, open day, performance, battle, trial, wedding, tea party, conflagration.</p>\n<p>References</p>\n<p>    * Dublin Core event </p>\n<p>[edit] The Event Model</p>\n<p>What follows is inspired by the framework described in \u00e2\u20ac\u0153Toward Common Event Model for Multimedia Applications\u00e2\u20ac\u009d</p>\n<p>In this framework, a common event model is proposed in order to allow integration and syndication of events and media from largely isolated multimedia applications and data sources, making possible novel cross-application and crossdata source multimedia services.</p>\n<p>The Event:</p>\n<p>&#8211; Is a significant occurrence that takes place in point in time.</p>\n<p>&#8211; Is a first-class entity that is independent on the media documenting it, that is, an event is not considered as a metadata describing a medium; instead, media are considered metadata documenting the course of an event.</p>\n<p>&#8211; Its model should provide means for global (that is, media-independent) unique identification. The event model\u00e2\u20ac\u2122s identification system permits the unique identification of events in a uniform fashion, regardless of the events\u00e2\u20ac\u2122 sources.</p>\n<p>&#8211; Has 6 basic aspects: spatial, temporal, informational, experiential, structural, causal</p>\n<p>The aspects of an event description are shortly described below. As an example, a &#8220;soccer match&#8221; event is considered.</p>\n<p>Temporal Aspect</p>\n<p>This aspect concerns the temporal awareness contained in an event description. There are different ways of capturing the temporal aspect:</p>\n<p>&#8211; Physical time: the instant or time interval when the event happens. It can be measured as a global time (e.g., date, time), a relative time (e.g., minutes of game) or a media-related time (e.g., frame numbers of a video documenting an event). The global and relative time can be modeled by the TimeInterval e-type previously described.</p>\n<p>&#8211; Logical time: application domain\u00e2\u20ac\u2122s abstract temporal concept which describes the event (e.g., the last match of the season)</p>\n<p>Spatial Aspect</p>\n<p>This aspect concerns the location awareness contained in an event description. There are different ways of capturing the spatial aspect:</p>\n<p>&#8211; Physical Location: the geographical place where the event takes place. A physical location is characterized by GPS coordinates, city and country specifications, etc. For example, the soccer match takes place at the Olimpic Stadium of Rome.</p>\n<p>&#8211; Logical Location: refers to a spatial domain concept (e.g., the match takes place in a stadium)</p>\n<p>Informational Aspect</p>\n<p>This aspect concerns information about the event. They can include the event type (e.g., soccer match), the event description (e.g., the Inter-Juve soccer match of the Champions League), the entities (agent and artifacts) involved (e.g., players and ball), the role of such entities (e.g., forward, goalkeeper, ball), other parameters (e.g., current score, number of players, etc.), the event &#8220;mode&#8221;, that is, how the event takes place: it coulbe an online event or a physical event.</p>\n<p>Experiential Aspect</p>\n<p>To augment the exploration and experience of an event, a media-aware component should be associated to it. A set of media (e.g., images, videos, audios, etc.) should thus documents how a (past) event evolved.</p>\n<p>Structural Aspect</p>\n<p>This aspect concerns the decomposition of high-level events into low-level events. Sub-events can be aggregated so to form more composite events (e.g., a set of matches forms a season). Knowledge on subevents can give useful hints on the composite event.</p>\n<p>Causal Aspect</p>\n<p>This aspect concerns the discovering of a chain of events which led to the event in question. Given an event, there should be an explicit representation of such cause-events chain.</p>\n<p>Event Relationships</p>\n<p>The last two aspects (Structural and Causal) put in relation more events. Here comes the issue of devising relationship between events. Some types of relationship are listed below:</p>\n<p>1) Structural</p>\n<p>Connect two events by expliciting a &#8220;part-of&#8221; relationship</p>\n<p>2) Causal</p>\n<p>Connect two events by expliciting a sort of &#8220;caused-by&#8221; relationship</p>\n<p>3) Temporal</p>\n<p>Connect two events by expliciting a temporal relationship (e.g., event A occurred during event B)</p>\n<p>4) Spatial</p>\n<p>Connect two events by expliciting a topological relation (e.g., event A takes place in the same place of event B)<br />\n[edit] Event Metadata</p>\n<p>In view of what has been said before, some metadata are proposed for the Event e-type. The string &#8220;//&#8221; represents a comment.</p>\n<p>Event<br />\n  {{Thing attributes},<br />\n   period TimeInterval, //Temporal Aspect<br />\n   logicalTime String,<br />\n   location Location,   //Spatial Aspect<br />\n   abstract String,   //Informational Aspect<br />\n   mode String,<br />\n   participant {Things},<br />\n   role {String},<br />\n   resources {Media}, //Experiential Aspect<br />\n   &#8230;<br />\n   }</p>\n<p>The above event metadata are possibily not complete.</p>\n<p>In LM:</p>\n<p>&#8211;
