\"manual\">\n\t  <p>Hobbes is a software package for efficiently mapping DNA\n\t  snippets (reads) against a reference DNA sequence. It can\n\t  map short and long reads, and supports Hamming distance\n\t  (only substitutions) and edit distance\n\t  (substitutions/insertions/deletions). Hobbes accepts both\n\t  single-end and paired-end reads for alignment, and can run\n\t  on multiple CPU cores using multithreading. It supports\n\t  three input formats (Fastq, Fasta, and text file) and two\n\t  output formats (<a\n\t  href=\"#bowtie-output-format\">Bowtie-like</a> format, <a\n\t  href=\"#sam-output-format\">SAM</a> format). Ambiguous bases\n\t  such as the 'N' character are treated as mismatches.  </p>\n</div>\n      \n<h3>System Requirements</h3>\n\t<div id=\"manual\">\n\t  <list>\n\t    <li>GNU Linux. We developed and tested Hobbes under Ubuntu.</li>\n\t    <li>GCC. Hobbes uses GCC builtin functions.</li>\n\t    <li>CMake. Required for compiling Hobbes (sudo apt-get install cmake).\n\t    <li>libbz2 and libz (sudo apt-get install libbz2-dev libbz2-dev).</li>\n\t    <li>Boost-iostreams (sudo apt-get install libboost-iostreams-dev).</li>\n\t    <li>A CPU supporting the <tt>popcount</tt> instruction (check your flags in /proc/cpuinfo).</li>\n\t  </list>\n\t</div>\n\n<h3>Compiling Hobbes</h3>\n\t<div id=\"manual\">\n\t  <list>\n\t    <li>Download the Hobbes source from <a href=\"download.shtml\">here</a>.</li>\n\t    <li>Extract the contents of the archive (tar -xf file).</li>\n\t    <li>cd into the Hobbes source root directory (hobbes1.3).</li>\n\t    <li>Run \"compile_16bitvector.sh\" to use 16 bits for Hobbes' bitvectors.</li>\n\t    <li>Run \"compile_32bitvector.sh\" to use 32 bits for Hobbes' bitvectors.</li>\n\t    <li>Alternatively, run \"cmake .\" followed by \"make\".</li>\n\t    <li>The Hobbes binaries are placed in the \"build\" directory.</li>\n\t  </list>\n\t</div>\n       \n\n<h3>Constructing a Hobbes Index</h3>\n<h4>Usage</h4>\n<tt>./hobbes-index --sref &lt;input fasta file&gt; -i &lt;output index file&gt;  -p &lt;number of threads&gt; -g &lt;gram length&gt;</tt>\n\n<h4>Example</h4>\n./hobbes-index --sref hg18.fa -i hg18.hix -p 4 -g 11\n\n<h4>Options</h4>\n\t<table width=\"100%\">\n\t  <tr height=40>\n\t    <td width=\"25%\"><tt>-g &lt;int&gt;</tt></td>\n\t    <td width=\"75%\">Use given gram length to build a Hobbes index. Each line has one read's mappings. Reads that are unmapped are also printed with appropriate flags set. Each line has the following tab separated fields:</p>\n       <p>\n       \n      \t<ol>\n       <li>Name of the read mapped.</li><br>\n       <li>SAM bitwise FLAG.  </li><br>\n       <li>RNAME: Reference sequence name of the mapping. If @SQ header lines are present, RNAME must be present in one of the SQ-SN tag. '*' if there is no mapping.</li><br>\n       <li>POS: 1-based leftmost mapping POSition of the first matching base. The first base in a reference sequence has coordinate 1. POS is set as 0 for an unmapped read.</li><br>\n       <li>MAPQ: Mapping Quality. A value 255 indicates that the mapping quality is not available. Since we don't support this yet, it's set to 255.</li><br>\n       <li>CIGAR: CIGAR string. <!--Currently unavailable and hence set to`*'.--></li><br>\n       <li>RNEXT: Reference sequence name of the NEXT fragment in the template.  Currently unavailable and hence set to`*'.</li><br>\n       <li>PNEXT: Position of the NEXT fragment in the template. In single-end alignment, it's set to 0; in paired-end alignment, it's the positon of it's mate pair.</li><br>\n       <li>TLEN: Signed insert size, it is set as 0 for single-end reads or when the information is unavailable</li><br>\n       <li>SEQ: Read sequence</li><br>\n       <li>QUAL: ASCII of base QUALity plus 33 (same as the quality string in the Sanger FASTQ format). Currently, the values are set only if the input file is a fastq file. Else, it is set to '*'.</li><br>\n       <li>Optional fields: Fields are tab-separated. For descriptions of all possible optional fields, see the SAM format specification. The fields relevant to hobbes are,</li><br>\n       <ol type=\"a\">\n       <li>NM:i:&lt;N&gt; - Mapped read has hamming/edit distance of &lt;N&gt;.</li><br>\n       <li>XA:Z: &lt;format&gt; - Alternative hits; &lt;format&gt;: (chr,pos,CIGAR,NM;)*, pos has a prefix of '+' if mapping is reported in forward reference strand, and '-'  for the reverse reference strand.</li><br>\n       </ol>\n       </ol>\n      \t</p>\n      </div> \n      \n      <h3>Bowtie Output Format</h3>\n      <div id=\"bowtie-output-format\">\n      <p>This format outputs one mapping per line.
