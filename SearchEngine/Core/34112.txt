Basic Authentication, it is inherently insecure, to wihch \n we can mitigate with a crytp/MD5, which also reduces the problem of\n storing my secure on another person's computer.\n \n But we should use Digest Authentication, which requires recoerable\n password on the server side. We do not want to transfer such secret\n in clear text, so we use Diffie-Hellman to generate such secrets.\n \n The problem is, it is a shared secret between a pair, so the pie can\n no longer just use one secret (my secret) to authenticate into everyone.\n Now the proxy has to use the many secrets to authenticate into every other\n person. This is acceptable, and maybe desirable, since we do not want \n to store our secrets into others. \n \n TODO: security: protocol: this needs a revisit once we are more familiar with
