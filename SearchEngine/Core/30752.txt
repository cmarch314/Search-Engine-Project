Handlers</A></H2>\r\n<P>\r\nAn error handler has 3 optional arguments.\r\nThe first argument is the number that identifies the error,\r\nthe second argument is\r\nthe culprit (a structure corresponding to the call which caused the\r\nerror).\r\nFor instance, if,\r\nsay, a type error occurs upon calling the third goal of the procedure\r\n<tt> p(2, Z)</tt>:\r\n<blockquote> <PRE> p(X, Y) :- a(X), b(X, Y), c(Y).\r\n</PRE>\r\n</blockquote>\r\nthe structure given to the error handler is <tt> b(2, Y)</tt>.\r\nNote that the handler could bind Y which would have the same effect\r\nas if <tt> b/2</tt> had done the binding.\r\n<P>\r\nThe third argument is only defined for a subset of the existing errors.\r\nIf the error occurred inside a tool body, it holds the caller module,\r\notherwise it is a free variable.\r\nNote that some events are not errors but are used for different purposes.\r\nIn thoses cases the second and third argument are sometimes used\r\ndifferently. See Appendix <A HREF=\"#chaperrors\"><IMG ALIGN=BOTTOM ALT=\"gif\" SRC=\"http://www.ecrc.de/eclipse/new_icons//cross_ref_motif.gif\"></A> for details.\r\n<P>\r\nThe error handler is free to ignore some of these arguments,\r\ni.e. it can have any arity from 0 to 3.\r\nThe first argument is provided for the case that the same procedure serves\r\nas the handler for several error types - then it can distinguish\r\nwhich is the actual error type.\r\n<A NAME=2534>&#160;</A>\r\nAn error handler is just an ordinary Prolog procedure<A NAME=tex2html553 HREF=\"footnode.html#2535\"><IMG ALIGN=BOTTOM ALT=\"gif\" SRC=\"http://www.ecrc.de/eclipse/new_icons//foot_motif.gif\"></A>,\r\nand thus within\r\nit a call may be made to any other procedure, or any built in predicate;\r\nthis in particular means that a call to <b> exit_block/1</b> may be\r\nmade (see the section on the <b> block/3</b>\r\npredicate).<A NAME=2538>&#160;</A> This will work\r\n'through' the call to the error handler, and so an exit may be made from\r\nwithin the handler out of the current block (i.e. back to the corresponding\r\ncall of the <b> block/3</b> predicate).\r\nSpecifying the predicates <b> true/0</b> or <b> fail/0</b> as error handlers\r\nwill make the erroneous predicate succeed (without binding\r\nany further variables) or fail respectively.\r\n<P>\r\n<BR> <HR><A NAME=tex2html4159 HREF=\"node160.html\"><IMG ALIGN=BOTTOM ALT=\"next\" SRC=\"http://www.ecrc.de/eclipse/new_icons//next_motif.gif\"></A>   <A NAME=tex2html4157 HREF=\"node157.html\"><IMG ALIGN=BOTTOM ALT=\"up\" SRC=\"http://www.ecrc.de/eclipse/new_icons//up_motif.gif\"></A>   <A NAME=tex2html4151 HREF=\"node158.html\"><IMG ALIGN=BOTTOM ALT=\"previous\" SRC=\"http://www.ecrc.de/eclipse/new_icons//previous_motif.gif\"></A>      <A NAME=tex2html4161 HREF=\"node321.html\"><IMG ALIGN=BOTTOM ALT=\"index\" SRC=\"http://www.ecrc.de/eclipse/new_icons//index_motif.gif\"></A>   <BR>\r\n<B> Next:</B> <A NAME=tex2html4160 HREF=\"node160.html\"> User Defined Errors</A>\r\n<B>Up:</B> <A NAME=tex2html4158 HREF=\"node157.html\"> Errors</A>\r\n<B> Previous:</B> <A NAME=tex2html4152 HREF=\"node158.html\"> Error Handlers</A>\r\n<BR> <HR> <P>\r\n<BR> <HR>\r\n<P><ADDRESS>\r\n<I>Micha Meier <BR>\r\nMon Mar  4 12:11:45 MET 1996</I>\r\n</ADDRESS>\r\n</BODY>\r\n", "id": 30752.0}
