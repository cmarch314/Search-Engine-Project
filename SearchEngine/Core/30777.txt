ECLiPSe</b></A>\r\n .\r\nIt accepts the syntax and semantics of the\r\npredicates <b> foreign/3, foreign_file/2 and load_foreign_files/2</b>\r\nof Quintus/SICStus.\r\nSince their external interface is incompatible with the <A NAME=tex2html1315 HREF=\"http://www.ecrc.de/eclipse/eclipse.html\"><b> ECLiPSe</b></A>\r\n  one,\r\nthis library generates a C source file which converts the <A foreign interface.\r\nNote that this approach uses more C code, but it is still more efficient\r\nthan using a generic procedure to check the argument\r\nof every external function.\r\n<P>\r\nAfter compiling definitions of the <b> foreign/3</b> predicate,\r\n(the predicate <b> foreign_file/2</b> is ignored),\r\nthe predicate <b> make_simple_interface</b> has to be called.\r\nThis predicate generates a file <i> interface.c</i>, which contains\r\nthe auxiliary C definitions.\r\nThis file has to be compiled to obtain the <i> interface.o</i> file,\r\nit might also have to be edited to include other <i> .h</i> files.\r\nAfter the file <i> interface.o</i> has been generated,\r\nthe system is fully compatible with the Quintus/SICStus foreign\r\ninterface, and calling <b> load_foreign_files/2</b> will connect\r\nthe external functions with <A NAME=tex2html1317 HREF=\"http://www.ecrc.de/eclipse/eclipse.html\"><b> ECLiPSe</b></A>\r\n .\r\n<P>\r\nAlthough it is possible to use this library to interface\r\nexisting independent C functions, its main use is to\r\nport foreign interface from Quintus or SICStus.\r\nPlease refer to their manuals for the description of
