and a value both stored as strings. They\ncan be attached to any of Chimera's hypermedia concepts. Typically, viewers\nuse the attribute mechanism to store application-specific information with\na concept to manipulate at a later time. This frees the viewer from having\nto store this information separately.</P>\n\n<P>For instance, links and anchors can be named by associating a <CODE>Name</CODE>\nattribute with them. The name can then be used to display the concept in\nlists in a more user-friendly manner. Another example, is using attributes\non anchors to store the information needed to reconstruct them for a particular\nview. For instance, a rectangle on an image can be recreated if its top-left\nand bottom-right coordinates are stored as attributes on the anchor which\nrepresents the rectangle.</P>\n\n<H2>Developer Information</H2>\n\n<P>The available operations on attributes are listed below.
