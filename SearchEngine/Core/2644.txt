Pregelix?</h2>\r\n<p align=\"justify\">Pregelix is an open-source implementation of the bulk-synchronous vertex-oriented programming model (<a href=\"http://googleresearch.blogspot.com/2009/06/large-scale-graph-computing-at-google.html\">Pregel API</a>) for large-scale graph analytics. Pregelix is based on an iterative dataflow design that is better tuned to handle both in-memory and out-of-core workloads efficiently. Pregelix supports a variety of physical runtime choice which can fit different sorts of graph algorithms, datasets, and clusters. Our motto is <strong>\"Big Graph Analytics Anywhere!\"</strong> --- from a single laptop to large enterprise clusters.</p>\r\n\r\n\r\n<h2><a name=\"Pregelix_quick_example:\"></a>Quick example:</h2>\r\n<p>\r\n<pre class=\"prettyprint lang-java\">\r\n  public class PageRankVertex extends Vertex&lt;VLongWritable, DoubleWritable, FloatWritable, \r\n\t\tDoubleWritable&gt; {\r\n\u00a0 \u00a0 ........\r\n\u00a0 \u00a0 @Override\r\n\u00a0 \u00a0 public void compute(Iterator&lt;DoubleWritable&gt; msgIterator) {\r\n\u00a0 \u00a0 \u00a0 \u00a0 .......\r\n\u00a0 \u00a0 \u00a0 \u00a0 sum = 0;\r\n\u00a0 \u00a0 \u00a0 \u00a0 while (msgIterator.hasNext()) {\r\n\u00a0 \u00a0 \u00a0 \u00a0 \u00a0 sum += msgIterator.next().get();\r\n\u00a0 \u00a0 \u00a0 \u00a0 }\r\n\u00a0 \u00a0 \u00a0 \u00a0 setVertexValue((0.15 / getNumVertices()) + 0.85 * sum);\r\n\u00a0 \u00a0 \u00a0 \u00a0 sendMsgToAllNeighbors(vertexValue / getEdges().size());\r\n\u00a0 \u00a0 \u00a0 \u00a0 ....\r\n\u00a0 \u00a0 }\r\n  }\r\n</pre>\r\n</p>\r\n<p algin=\"justify\">The above code is the\u00a0<strong>PageRank</strong>\u00a0implementation on Pregelix.</p>\r\n\r\n\r\n<h2><a name=\"Pregelix_performance:\"></a>Performance:</h2>\r\n<p align=\"justify\">\r\nWe have compared Pregelix with several popular Big Graph Analytics platforms, including Giraph, Hama, GraphLab, and GraphX.
