These conformers are managed in\na manner very similar to atoms and bonds.  Conformers can only be created\nor destroyed in the context of an OEMol, and must be accessed via member\nfunctions laid out in the OEMCMolBaseT API.\n\n<P>\nConformers are represented by the abstract base-class OEConfBaseT (again the T\nindicates this is a template class).  It is sometimes convenient to be able\nto treat a conformer object as its own single-conformer molecule.  For this\nreason, OEConfBaseT inherits from OEMolBase.  Thus, thought a conformer is\ncontained within a multi-conformer molecule, it can act as a single-conformer\nmolecule, and can be passed to functions which have an OEMolBase argument.\n\n<P>\n<dl><dd><pre class=\"verbatim\">Simplistic OEChem inheritance scheme:\n                OEBase\n                  |\n                  |\n                OEMolBase-------------------\n                  |                        |\n                  |                        |\n                OEConfBaseTo-----------&gt;OEMCMolBaseT\n&lt;-- indicates inheritance\no-&gt; indicates object o is contained int the object indicated by &gt;\n</pre></dl>\n\n<P>\nOne must be cautious when utilizing this OEMolBase inheritance functionality.\nEach multi-conformer molecule has only a single heavy-atom graph.  For\nfunctions which query this graph portion of a molecule, a conformer will\nreflect the graph properties of its parent multi-conformer molecule.  It is\nonly independent of its parent for non-graph (<I>e.g.</I> conformational)\nproperties.\nThe logical extension of this principle is that changes made to the\n<i>graph</i> properties of one conformer will effect its parent multi-conformer\nmolecule and thus all the other conformers in that molecule as well.\n\n<P>\n\n<DIV CLASS=\"navigation\">\n<p><hr>\n<table align=\"center\" border=\"0\" width=\"100%\" cellpadding=\"0\" cellspacing=\"0\">\n<tr>\n<td rowspan=2><A HREF=\"node104.html\"><img src=\"../icons/previous.gif\"\n  border=\"0\" alt=\"Previous Page\"></A></td>\n<td rowspan=2><A HREF=\"node102.html\"><img src=\"../icons/up.gif\"\n  border=\"0\" alt=\"Up One Level\"></A></td>\n<td rowspan=2><A HREF=\"node106.html\"><img src=\"../icons/next.gif\"\n  border=\"0\" alt=\"Next Page\"></A></td><td width=\"100%\" rowspan=\"2\"> </td>\n<td valign=\"center\" align=\"right\" height=\"56\" background=\"../icons/title_bkgd.gif\" colspan=4 class=\"navigation\">OEChem - C++ Theory Manual\n<BR>\nVersion 1.3.1</td></tr>\n<tr><td><img src=\"../icons/spacer_1.gif\"\n  border=\"0\" alt=\"\"></td>\n<td><A href=\"contents.html\"><img src=\"../icons/contents.gif\"\n  border=\"0\" alt=\"Contents\"></A></td>\n<td><img src=\"../icons/spacer_2.gif\"\n  border=\"0\" alt=\"\"></td>\n<td><img src=\"../icons/blank_4.gif\"\n  border=\"0\" alt=\"\"></td>\n</tr></table>\n<b class=\"navlabel\">Previous:</b> <a class=\"sectref\" HREF=\"node104.html\">8.2 Multi-conformer and single-conformer</A>\n<b class=\"navlabel\">Up:</b> <a class=\"sectref\" HREF=\"node102.html\">8.
