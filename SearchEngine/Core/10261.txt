setup</a></h3>\n<div class=\"level3\">\n\n<p>\nAt the very least you will need to tell git who you are. The following two commands will set your name and email address. This information will show up in the commit log.\n\n</p>\n<pre class=\"code bash\">git config <span class=\"re5\">--global</span> user.name <span class=\"st0\">&quot;Your Name&quot;</span>\ngit config <span class=\"re5\">--global</span> user.email <span class=\"st0\">&quot;your_email@whatever.edu&quot;</span></pre>\n\n<p>\n\nIf you have <a href=\"http://en.gravatar.com/\" class=\"urlextern\" title=\"http://en.gravatar.com/\"  rel=\"nofollow\">Gravatar</a> account, the associated image will appear in many git <acronym title=\"Graphical User Interface\">GUI</acronym> programs.\n</p>\n\n</div>\n<!-- SECTION \"Mandatory setup\" [169-602] -->\n<h3><a name=\"ignoring_files\" id=\"ignoring_files\">Ignoring files</a></h3>\n<div class=\"level3\">\n\n<p>\nSome files you never want in your repository. This includes LaTeX auxiliary files, pdf files compiled from TeX sources, and operating system trash. You can activate a global ignore file with the command\n\n</p>\n<pre class=\"code bash\">git config <span class=\"re5\">--global</span> core.excludesfile <span class=\"st0\">&quot;~/.gitignore_global&quot;</span></pre>\n\n<p>\n\nIf you are using <acronym title=\"Operating System\">OS</acronym> X, then you may have a global ignore file that looks like\n\n</p>\n<dl class=\"code\">\n<dt><a href=\"/_export/code/wiki:git_tutorial?codeblock=2\" title=\"Download Snippet\" class=\"mediafile mf_gitignore_global\">.gitignore_global</a></dt>\n<dd><pre class=\"code\">\n.DS_Store\n.AppleDouble\n.LSOverride\nIcon\n._*\n.Spotlight-V100\n.Trashes\n</pre>\n</dd></dl>\n\n<p>\n\nAnd, if you are using Windows, then you may have a global ignore file that looks like\n\n</p>\n<dl class=\"code\">\n<dt><a href=\"/_export/code/wiki:git_tutorial?codeblock=3\" title=\"Download Snippet\" class=\"mediafile mf_gitignore_global\">.gitignore_global</a></dt>\n<dd><pre class=\"code\">\nThumbs.db\nehthumbs.db\nDesktop.ini\n$RECYCLE.BIN/\n</pre>\n</dd></dl>\n\n<p>\n\nThe problem with using global ignore file is that it requires all of your coauthors to have setup a global ignore file. If they have not you will still end up with crud in your repository. For this reason a better solution is a per repository ignore file. If you add file with name <code>.gitignore</code> to the top level of your repository, then all files matching the patterns in this file will be ignored. A good general purpose ignore file for LaTeX documents is\n\n</p>\n<dl class=\"code\">\n<dt><a href=\"/_export/code/wiki:git_tutorial?codeblock=4\" title=\"Download Snippet\" class=\"mediafile mf_gitignore\">.gitignore</a></dt>\n<dd><pre class=\"code\">\npaper.pdf\n\n##### LaTeX Files #####\n*.aux\n*.bbl\n*.blg\n*.dvi\n*.fdb_latexmk\n*.glg\n*.glo\n*.gls\n*.idx\n*.ilg\n*.ind\n*.ist\n*.lof\n*.log\n*.lot\n*.nav\n*.nlo\n*.out\n*.pdfsync\n*.ps\n*.snm\n*.synctex.gz\n*.toc\n*.vrb\n*.maf\n*.mtc\n*.mtc0\n\n##### OS X Files #####\n.DS_Store\n.AppleDouble\n.LSOverride\nIcon\n._*\n.Spotlight-V100\n.Trashes\n\n##### Windows Files #####\nThumbs.db\nehthumbs.db\nDesktop.ini\n$RECYCLE.BIN/\n</pre>\n</dd></dl>\n\n<p>\n\nThe top line of this file should be changed to match the output of pdflatex.\n</p>\n\n</div>\n<!-- SECTION \"Ignoring files\" [603-2210] -->\n<h2><a name=\"git_workflow\" id=\"git_workflow\">Git workflow</a></h2>\n<div class=\"level2\">\n\n</div>\n<!-- SECTION \"Git workflow\" [2211-2236] -->\n<h3><a name=\"simple_workflow\" id=\"simple_workflow\">Simple workflow</a></h3>\n<div class=\"level3\">\n\n<p>\nThe first step is to clone the repository you will be working in. To add the file <code>foo.bar</code> to the repository use the following command:\n\n</p>\n<pre class=\"code bash\">git add foo.bar</pre>\n\n<p>\nWhen you have edited files to the point where you are ready to \u201ccheck in\u201d your changes, you can use can use the following command to commit all changes.\n\n</p>\n<pre class=\"code bash\">git commit <span class=\"re5\">-a</span></pre>\n\n<p>\nIn git commits are only to your local repository. To share your local commits with your coauthors you use the push command.\n\n</p>\n<pre class=\"code bash\">git push</pre>\n\n<p>\nOn the first push to an empty repository you will get the error:\n\n</p>\n<pre class=\"code\">\nNo refs in common and none specified; doing nothing.\nPerhaps you should specify a branch such as &#039;master&#039;.\nfatal: The remote end hung up unexpectedly\nerror: failed to push some refs to &#039;&lt;repo name&gt;&#039;\n</pre>\n\n<p>\n\nFor the first push you will need to specify the full branch name with the command:\n\n</p>\n<pre
