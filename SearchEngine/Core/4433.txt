known security hole.&#160; Therefore, you should\nturn off any unneeded services and ensure any services you run are the \nlatest\nversion and are properly configured.\n          </p>\n<ul>\n<li> Services in <code>/etc/inetd.conf</code>\n<ul>\n<li> <code>Inetd</code> is the daemon that manages many \nsystem\n                services.&#160; These are listed in its configuration file, <code>/etc/inetd.conf</code>,\n                which can easily be modified to prevent <code>inetd</code>\n from\n                starting unwanted services.&#160; If you are the only user of\n                your workstation, you can have almost everything \ncommented out and will most likely not notice their absence.</li>\n<li> Services which are often unneeded include: <code>echo</code>,\n <code>daytime</code>, <code>chargen</code>, <code>time</code>, <code>login</code>,\n <code>shell</code>,\n                and <code>exec</code>.&#160; To turn these off, follow the \ninstructions\n                at <a href=\"http://www.oit.uci.edu/support/security/turning-off-inetd-service.html\">\n Turning\n                off an <i>inetd</i>-launched service</a>.</li>\n<li> Services you probably want to keep include: <code>telnet</code>,\n <code>ftp</code>, <code>identd</code>.</li>\n</ul>\n</li>\n<li> Services started from system start-up scripts</li>\n<p> Many other services are started in system start-up \nscripts. You\n              should review your start-up scripts and comment out ones \nwhich\n              are not appropriate for your workstation's purpose. For \nexample, <code>rpc.mountd</code> and <code>rpc.nfsd</code> are\n              probably unneeded on a single user workstation. <i>kill -9</i>\n any\n              unwanted services and then comment them out in the \nappropriate\n              system start-up file.            </p>\n<li> Other services\n<ul>\n<li> To determine what other services you are running, use\n the\n                command:\n<pre>          netstat -a | more</pre>\n</li>\n<li> Use your package manager to list the packages \ninstalled (<code>dpkg</code> for\n                Debian and <code>rpm</code> for RedHat).</li>\n<li> Unless your machine is acting as a mail server, you \nshould <a href=\"http://www.oit.uci.edu/support/dcs/security/turning-off-sendmail.html\">\n turn\n                  off sendmail</a>  since it is very large and is one of\n the\n                  most common targets for attacks because of its \nnumerous security\n                  holes. Note: <code>sendmail</code> is not necessary if\n you\n                  are using <code>pop</code> to read your mail. </li>\n<li> If you are not actively sharing files or printers from\n your\n                machine with machines running a Microsoft operating \nsystem, you\n                should <a href=\"http://www.oit.uci.edu/support/dcs/security/turning-off-samba.html\">\n turn\n                off samba</a>.</li>\n</ul>\n</li>\n<li> Do not run NIS or NIS+ if you do not need them. A \nstandalone machine\n              will never need these.\n<ul>\n<li> Define each netgroup to contain only usernames or \nonly hostnames.&#160; All\n                utilities parse <code>/etc/netgroup</code> for either \nhosts or\n                usernames, but never both.&#160; Using separate netgroups \nmakes\n                it easier to remember the function of each netgroup. The\n added\n                time required to administer these extra netgroups is a \nsmall\n                cost in ensuring that strange permission combinations \nwill not leave your machine in an insecure state.</li>\n</ul>\n</li>\n<li> Web servers: If you are running a web server, make sure\n you\n              have the latest available release.&#160; The latest version of\n              Apache can be obtained from the official <a href=\"http://www.apache.org/dist/\">Apache\n              Distribution</a> site.</li>\n<li> Anonymous FTP\n<ul>\n<li> Make sure that you are running the most recent \nversion of <code>ftpd</code>.\n                There are several well-known security problems with the \nones\n                that ship on distribution media.</li>\n<li> Check your anonymous FTP configuration. It is \nimportant to\n                follow the instructions provided with the operating \nsystem to\n                properly configure the files and directories available \nthrough\n                anonymous FTP (for example, file and directory \npermissions, ownership\n                and group).</li>\n<li> You should not use your system's standard password \nfile or\n                group file as the password file or group file for FTP.</li>\n<li> The anonymous FTP root directory and its two \nsubdirectories, <code>etc</code> and <code>bin</code>,\n                should not be owned by the FTP user.</li>\n</ul>\n</li>\n</ul>\n\n\n</div>\n</blockquote>\n</div>\n<div id=\"sidebar\"></div>\n<div id=\"footer\"><a href=\"http://www.uci.edu/copyright.php\">Copyright Inquiries</a> |\n   <a href=\"http://www.uci.edu/cgi-bin/phonebook\">UCI Directory</a> |\n   <a href=\"http://intranet.ics.uci.edu/\">Intranet</a> |\n   \n   <!-- <a href=\"http://www.ics.uci.edu/sitemap.php\">Site Map</a> | -->\n \n  <a href=\"mailto: i%63%73%77%65%62m%61s%74%65r@%69c%73%2e%75c%69%2eedu\">icswebmaster</a> |\nUpdated: \nNovember 05 2013</div>\n</div>\n<script type=\"text/javascript\">\nvar gaJsHost = ((\"https:\" == document.location.protocol) ? \"https://ssl.\" : \"http://www.\");\ndocument.write(unescape(\"%3Cscript src='\" + gaJsHost + \"google-analytics.com/ga.js' type='text/javascript'%3E%3C/script%3E\"));\n</script>\n<script type=\"text/javascript\">\nvar pageTracker = _gat._getTracker(\"UA-605898-1\");\npageTracker._trackPageview();\n</script>\n</body>\n</html>", "id": 4433.0}
