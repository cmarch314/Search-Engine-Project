of <A HREF=\"anchors.html\">anchors</A>. All of the other\nconcepts discussed in this overview provide the foundation for the power\nof Chimera links. Since anchors are associated only at a conceptual level\nwith <A HREF=\"objects.html\">objects</A> and <A HREF=\"views.html\">views</A>,\nlinks can forge paths between a wide range of information. Links can contain\nany number of anchors. Viewers can directly create links and anchors to\nthem using the Chimera API. The end-user authors links by employing the\nfollowing process.</P>\n\n<UL>\n  <LI>Using a link editor, the user creates a new empty link.\n  <LI>The user selects the new link to make it <I>active</I>. The active\n  link is shared between all of a user's clients.\n  <LI>The user can then add anchors to the active link from any active client\n  by selecting the appropriate anchor and invoking the <CODE>Add_Anchor_To_Active_Link</CODE>\n  operation. Almost all Chimera clients provide access to this operation\n  in some fashion, typically by using a pull-down or pop-up menu.\n</UL>\n\n<P>The default link traversal semantics for Chimera is to send a link traversal\nevent to all clients which manage the destination anchors of the link. Developers,\nhowever, can customize the semantics of link traversal for an application,\nor suite of applications. For more information, see the <A HREF=\"../../dev/docs/index.html\">on-line\ndocumentation</A>.</P>\n\n<H2>Developer Information</H2>\n\n<P>The available operations on links are listed below.
