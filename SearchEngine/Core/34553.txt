are even some pages that are still talking about Trac 8 Installing Trac Using easy install One way to install Trac is using setuptools With setuptools you can install Trac from the subversion repository A few examples first install of the latest stable version Trac 12 1 with i18n support easy install Babel 9 5 Genshi 6 easy install Trac It s very important to run the two easy install commands separately otherwise the message catalogs won t be generated upgrade to the latest stable version of Trac easy install U Trac upgrade to the latest trunk development version 13dev easy install U Trac dev For upgrades reading the TracUpgrade page is mandatory of course From source If you want more control you can download the source in archive form or do a checkout from one of the official Trac TracRepositories source code repositories Be sure to have the prerequisites already installed You can also obtain the Genshi and Babel source packages from http www edgewall org and follow for them a similar installation procedure or you can just easy install those see above Once you ve unpacked the Trac archive or performed the checkout move in the top level folder and do python setup py install You ll need root permissions or equivalent for this step This will byte compile the python source code and install it as an egg file or folder in the site packages directory of your Python installation The egg will also contain all other resources needed by standard Trac such as htdocs and templates The script will also install the trac admin command line tool used to create and maintain project environments as well as the tracd standalone server If you install from source and want to make Trac available in other languages make sure Babel is installed Only then perform the install or simply redo the install once again afterwards if you realize Babel was not yet installed python setup py install Alternatively you can do a bdist egg and copy the egg from dist to the place of your choice or you can create a Windows installer bdist wininst Advanced Options To install Trac to a custom location or find out about other advanced installation options run easy install help Also see Installing Python Modules for detailed information Specifically you might be interested in easy install prefix path to installdir or if installing Trac to a Mac OS X system easy install prefix usr local install dir Library Python 2 5 site packages Note If installing on Mac OS X 1 6 running easy install http svn edgewall org repos trac trunk will install into usr local and Library Python 2 6 site packages by default The above will place your tracd and trac admin commands into usr local bin and will install the Trac libraries and dependencies into Library Python 2 5 site packages which is Apple s preferred location for third party Python application installations Using pip pip is an easy install replacement that is very useful to quickly install python packages To get a trac installation up and running in less than 5 minutes Assuming you want to have your entire pip installation in opt user trac pip E opt user trac install trac psycopg2 or pip E opt user trac install trac mysql python Make sure your OS specific headers are available for pip to automatically build PostgreSQL libpq dev or MySQL libmysqlclient dev bindings pip will automatically resolve all dependencies like Genshi pygments etc and download the latest packages on pypi python org and create a self contained installation in opt user trac All commands tracd trac admin are available in opt user trac bin This can also be leveraged for mod python using PythonHandler directive and mod wsgi using WSGIDaemonProcess directive Additionally you can install several trac plugins listed here through pip Creating a Project Environment A Trac environment is the backend storage where Trac stores information like wiki pages tickets reports settings etc An environment is basically a directory that contains a human readable configuration file and various other files and directories A new environment is created using trac admin trac admin path to myproject initenv trac admin will prompt you for the information it needs to create the environment such as the name of the project and the database connection string If you re not sure what to specify for one of these options just press Enter to use the default value Using the default database connection string in particular will always work as long as you have SQLite installed For the other database backends you should plan ahead and already have a database ready to use at this point Since 12 Trac doesn t ask for a source code repository anymore when creating an environment Repositories can be added afterward or the version control support can be disabled completely if you don t need it Also note that the values you specify here can be changed later by directly editing the conf trac ini configuration file Finally make sure the user account under which the web front end runs will have write permissions to the environment directory and all the files inside This will be the case if you run trac admin initenv as this user If not you should set the correct user afterwards For example on Linux with the web server running as user apache and group apache enter chown R apache apache path to myproject Warning Please only use ASCII characters for account name and project path unicode characters are not supported there Running the Standalone Server After having created a Trac environment you can easily try the web interface by running the standalone server tracd tracd port 8 path to myproject Then fire up a browser and visit http localhost 8 You should get a simple listing of all environments that tracd knows about Follow the link to the environment you just created and you should see Trac in action If you only plan on managing a single project with Trac you can have the standalone server skip the environment list by starting it like this tracd s port 8 path to myproject Running Trac on a Web Server Trac provides various options for connecting to a real web server CGI FastCGI mod wsgi and mod python For decent performance it is recommended that you use either FastCGI or mod wsgi Trac also supports AJP which may be your choice if you want to connect to IIS Generating the Trac cgi bin directory In order for Trac to function properly with FastCGI you need to have a trac fcgi file and for mod wsgi a trac wsgi file These are Python scripts which load the appropriate Python code They can be generated using the deploy option of trac admin There is however a bit of a chicken and egg problem The trac admin command requires an existing environment to function but complains if the deploy directory already exists This is a problem because environments are often stored in a subdirectory of the deploy The solution is to do something like this mkdir p usr share trac projects my project trac admin usr share trac projects my project initenv trac admin usr share trac projects my project deploy tmp deploy mv tmp deploy usr share trac Setting up the Plugin Cache Some Python plugins need to be extracted to a cache directory By default the cache resides in the home directory of the current user When running Trac on a Web Server as a dedicated user which is highly recommended who has no home directory this might prevent the plugins from starting To override the cache location you can set the PYTHON EGG CACHE environment variable Refer to your server documentation for detailed instructions on how to set environment variables Configuring Authentication The process of adding removing and configuring user accounts for authentication depends on the specific way you run Trac The basic procedure is described in the Adding Authentication section on the TracCgi page To learn how to setup authentication for the frontend you re using please refer to one of the following pages TracStandalone if you use the standalone server tracd TracCgi if you use the CGI or FastCGI web front ends TracModWSGI if you use the Apache mod wsgi web front end TracModPython if you use the Apache mod python web front end Automatic reference to the SVN changesets in Trac tickets You can configure SVN to automatically add a reference to the changeset into the ticket comments whenever changes are committed to the repository The description of the commit needs to contain one of the following formulas Refs 123 to reference this changeset in 123 ticket Fixes 123 to reference this changeset and close 123 ticket with the default status fixed This functionality requires a post commit hook to be installed as described in TracRepositoryAdmin and enabling the optional commit updater components by adding the following line to the components section of your trac ini or enabling the components in the Plugins admin panel tracopt ticket commit updater enabled For more information see the documentation of the CommitTicketUpdater component in the Plugins admin panel Using Trac Once you have your Trac site up and running you should be able to create tickets view the timeline browse your version control repository if configured etc Keep in mind that anonymous not logged in users can by default access most but not all of the features You will need to configure authentication and grant additional permissions to authenticated users to see the full set of features Enjoy The Trac Team See also TracInstallPlatforms TracGuide TracCgi TracFastCgi install Trac, the following software packages must be installed:\n</p>\n<ul><li><a class=\"ext-link\" href=\"http://www.python.org/\"><span class=\"icon\">\u200b</span>Python</a>, version &gt;= 2.4 and &lt; 3.0\n(note that we dropped the support for Python 2.3 in this release)\n</li><li><a class=\"ext-link\" href=\"http://peak.telecommunity.com/DevCenter/setuptools\"><span class=\"icon\">\u200b</span>setuptools</a>, version &gt;= 0.6\n</li><li><a class=\"ext-link\" href=\"http://genshi.edgewall.org/wiki/Download\"><span class=\"icon\">\u200b</span>Genshi</a>, version &gt;= 0.6\n</li></ul><p>\nYou also need a database system and the corresponding python bindings.\nThe database can be either SQLite, PostgreSQL or MySQL.\n</p>\n<h4 id=\"ForSQLite\">For the SQLite database</h4>\n<p>\nIf you're using Python 2.5 or 2.6, you already have everything you need.\n</p>\n<p>\nIf you're using Python 2.4 and need pysqlite, you can download from \n<a class=\"ext-link\" href=\"http://code.google.com/p/pysqlite/downloads/list\"><span class=\"icon\">\u200b</span>google code</a> the Windows installers or the tar.gz archive for building from source: \n</p>\n<pre class=\"wiki\">$ tar xvfz &lt;version&gt;.tar.gz \n$ cd &lt;version&gt; \n$ python setup.py build_static install \n</pre><p>\n \nThis will extract the SQLite code and build the bindings. \n</p>\n<p>\nTo install SQLite, your system may require the development headers. Without these you will get various GCC related errors when attempting to build:\n</p>\n<pre class=\"wiki\">$ apt-get install libsqlite3-dev\n</pre><p>\nSQLite 2.x is no longer supported, and neither is PySqlite 1.1.x.\n</p>\n<p>\nA known bug PySqlite versions 2.5.2-4 prohibits upgrade of trac databases\nfrom 0.11.x to 0.12. Please use versions 2.5.5 and newer or 2.5.1 and\nolder. See <a class=\"ext-link\" href=\"http://trac.edgewall.org/intertrac/%239434\" title=\"#9434 in Trac project trac\"><span class=\"icon\">\u200b</span>#9434</a> for more detail.\n</p>\n<p>\nSee additional information in <a class=\"ext-link\" href=\"http://trac.edgewall.org/intertrac/PySqlite\" title=\"PySqlite in Trac project trac\"><span class=\"icon\">\u200b</span>PySqlite</a>.\n</p>\n<h4 You can also obtain the Genshi and Babel source packages from <a class=\"ext-link\" href=\"http://www.edgewall.org\"><span class=\"icon\">\u200b</span>http://www.edgewall.org</a> and follow for them a similar installation procedure, or you can just easy_install those, see <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracInstall#Usingeasy_install\">above</a>.\n</p>\n<p>\nOnce you've unpacked the Trac archive or performed the checkout, move in the top-level folder and do:\n</p>\n<pre class=\"wiki\">$ python ./setup.py install\n</pre><p>\n<em>You'll need root permissions or equivalent for this step.</em>\n</p>\n<p>\nThis will byte-compile the python source code and install it as an .egg file or folder in the <tt>site-packages</tt> directory\nof your Python installation. The .egg will also contain all other resources needed by standard Trac, such as htdocs and templates.\n</p>\n<p>\nThe script will also install the <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracAdmin\">trac-admin</a> command-line tool, used to create and maintain <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracEnvironment\">project environments</a>, as well as the <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracStandalone\">tracd</a> standalone server.\n</p>\n<p>\nIf you install from source and want to make Trac available in other languages, make sure  Babel is installed. Only then, perform the <tt>install</tt> (or simply redo the <tt>install</tt> once again afterwards if you realize Babel was not yet installed):\n</p>\n<pre class=\"wiki\">$ python ./setup.py install\n</pre><p>\nAlternatively, you can do a <tt>bdist_egg</tt> and copy the .egg from dist/ to the place of your choice, or you can create a Windows installer (<tt>bdist_wininst</tt>).\n</p>\n<h3 id=\"AdvancedOptions\">Advanced Options</h3>\n<p>\nTo install Trac to a custom location, or find out about other advanced installation options, run:\n</p>\n<pre class=\"wiki\">easy_install --help\n</pre><p>\nAlso see <a class=\"ext-link\" href=\"http://docs.python.org/inst/inst.html\"><span class=\"icon\">\u200b</span>Installing Python Modules</a> for detailed information.\n</p>\n<p>\nSpecifically, you might be interested in:\n</p>\n<pre class=\"wiki\">easy_install --prefix=/path/to/installdir\n</pre><p>\nor, if installing Trac to a Mac OS X system:\n</p>\n<pre class=\"wiki\">easy_install --prefix=/usr/local --install-dir=/Library/Python/2.5/site-packages\n</pre><p>\nNote: If installing on Mac OS X 10.6 running <tt> easy_install http://svn.edgewall.org/repos/trac/trunk </tt> will install into <tt> /usr/local </tt> and <tt> /Library/Python/2.6/site-packages </tt> by default\n</p>\n<p>\nThe above will place your <tt>tracd</tt> and <tt>trac-admin</tt> commands into <tt>/usr/local/bin</tt> and will install the Trac libraries and dependencies into <tt>/Library/Python/2.5/site-packages</tt>, which is Apple's preferred location for third-party Python application installations.\n</p>\n<h3 id=\"Usingpip\">Using <tt>pip</tt></h3>\n<p>\n'pip' is an easy_install replacement that is very useful to quickly install python packages.\nTo get a trac installation up and running in less than 5 minutes:\n</p>\n<p>\nAssuming you want to have your entire pip installation in /opt/user/trac:\n</p>\n<ul><li>\n<pre class=\"wiki\">pip -E /opt/user/trac install trac psycopg2 \n</pre></li></ul><p>\nor\n</p>\n<ul><li>\n<pre class=\"wiki\">pip -E /opt/user/trac install trac mysql-python \n</pre></li></ul><p>\nMake sure your OS specific headers are available for pip to automatically build PostgreSQL (libpq-dev) or MySQL (libmysqlclient-dev) bindings.\n</p>\n<p>\npip will automatically resolve all dependencies (like Genshi, pygments, etc.) and download the latest packages on pypi.python.org and create a self contained installation in /opt/user/trac .\n</p>\n<p>\nAll commands (tracd, trac-admin) are available in /opt/user/trac/bin. This can also be leveraged for mod_python (using PythonHandler directive) and mod_wsgi (using WSGIDaemonProcess directive)\n</p>\n<p>\nAdditionally, you can install several trac plugins (listed <a class=\"ext-link\" href=\"http://pypi.python.org/pypi?:action=search&amp;term=trac&amp;submit=search\"><span class=\"icon\">\u200b</span>here</a>) through pip.\n</p>\n<h2 id=\"CreatingaProjectEnvironment\">Creating a Project Environment</h2>\n<p>\nA <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracEnvironment\">Trac environment</a> is the backend storage where Trac stores information like wiki pages, tickets, reports, settings, etc. An environment is basically a directory that contains a human-readable <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracIni\">configuration file</a>, and various other files and directories.\n</p>\n<p>\nA new environment is created using <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracAdmin\">trac-admin</a>:\n</p>\n<pre class=\"wiki\">$ trac-admin /path/to/myproject initenv\n</pre><p>\n<a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracAdmin\">trac-admin</a> will prompt you for the information it needs to create the environment, such as the name of the project and the <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracEnvironment#DatabaseConnectionStrings\">database connection string</a>. If you're not sure what to specify for one of these options, just press <tt>&lt;Enter&gt;</tt> to use the default value. \n</p>\n<p>\nUsing the default database connection string in particular will always work as long as you have SQLite installed.\nFor the other <a class=\"missing wiki\">database backends?</a> you should plan ahead and already have a database ready to use at this point.\n</p>\n<p>\nSince 0.12, Trac doesn't ask for a <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracEnvironment#SourceCodeRepository\">source code repository</a> anymore when creating an environment. Repositories can be <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracRepositoryAdmin\">added</a> afterward, or the version control support can be disabled completely if you don't need it.\n</p>\n<p>\nAlso note that the values you specify here can be changed later by directly editing the <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracIni\">conf/trac.ini</a> configuration file.\n</p>\n<p>\nFinally, make sure the user account under which the web front-end runs will have <strong>write permissions</strong> to the environment directory and all the files inside. This will be the case if you run <tt>trac-admin ... initenv</tt> as this user. If not, you should set the correct user afterwards. For example on Linux, with the web server running as user <tt>apache</tt> and group <tt>apache</tt>, enter:\n</p>\n<pre class=\"wiki\"># chown -R apache.apache /path/to/myproject\n</pre><div class=\"important\"><p>\n<strong>Warning:</strong> Please only use ASCII-characters for account name and project path, unicode characters are not supported there.\n</p>\n</div><h2 id=\"RunningtheStandaloneServer\">Running the Standalone Server</h2>\n<p>\nAfter having created a Trac environment, you can easily try the web interface by running the standalone server <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracStandalone\">tracd</a>:\n</p>\n<pre class=\"wiki\">$ tracd --port 8000 /path/to/myproject\n</pre><p>\nThen, fire up a browser and visit <tt>http://localhost:8000/</tt>. You should get a simple listing of all environments that <tt>tracd</tt> knows about. Follow the link to the environment you just created, and you should see Trac in action. If you only plan on managing a single project with Trac you can have the standalone server skip the environment list by starting it like this:\n</p>\n<pre class=\"wiki\">$ tracd -s --port 8000 /path/to/myproject\n</pre><h2 id=\"RunningTraconaWebServer\">Running Trac on a Web Server</h2>\n<p>\nTrac provides various options for connecting to a \"real\" web server: <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracCgi\">CGI</a>, <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracFastCgi\">FastCGI</a>, <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracModWSGI\">mod_wsgi</a> and <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracModPython\">mod_python</a>. For decent performance, it is recommended that you use either FastCGI or mod_wsgi.\n</p>\n<p>\nTrac also supports <a class=\"ext-link\" href=\"http://trac.edgewall.org/intertrac/TracOnWindowsIisAjp\" title=\"TracOnWindowsIisAjp in Trac project trac\"><span class=\"icon\">\u200b</span>AJP</a> which may be your choice if you want to connect to IIS.\n</p>\n<h4 id=\"GeneratingtheTraccgi-bindirectory\">Generating the Trac cgi-bin directory</h4>\n<p>\nIn order for Trac to function properly with FastCGI you need to have a <tt>trac.fcgi</tt> file and for mod_wsgi a <tt>trac.wsgi</tt> file. These are Python scripts which load the appropriate Python code. They can be generated using the <tt>deploy</tt> option of <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracAdmin\">trac-admin</a>.\n</p>\n<p>\nThere is, however, a bit of a chicken-and-egg problem. The <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracAdmin\">trac-admin</a> command requires an existing environment to function, but complains if the deploy directory already exists. This is a problem, because environments are often stored in a subdirectory of the deploy. The solution is to do something like this:\n</p>\n<pre class=\"wiki\">mkdir -p /usr/share/trac/projects/my-project\ntrac-admin /usr/share/trac/projects/my-project initenv\ntrac-admin /usr/share/trac/projects/my-project deploy /tmp/deploy\nmv /tmp/deploy/* /usr/share/trac\n</pre><h4 id=\"SettingupthePluginCache\">Setting up the Plugin Cache</h4>\n<p>\nSome Python plugins need to be extracted to a cache directory. By default the cache resides in the home directory of the current user. When running Trac on a Web Server as a dedicated user (which is highly recommended) who has no home directory, this might prevent the plugins from starting. To override the cache location you can set the PYTHON_EGG_CACHE environment variable. Refer to your server documentation for detailed instructions on how to set environment variables.\n</p>\n<h2 id=\"ConfiguringAuthentication\">Configuring Authentication</h2>\n<p>\nThe process of adding, removing, and configuring user accounts for authentication depends on the specific way you run Trac. The description of the commit needs to contain one of the following formulas:\n</p>\n<ul><li><strong><tt>Refs #123</tt></strong> - to reference this changeset in <tt>#123</tt> ticket\n</li><li><strong><tt>Fixes #123</tt></strong> - to reference this changeset and close <tt>#123</tt> ticket with the default status <em>fixed</em>\n</li></ul><p>\nThis functionality requires a post-commit hook to be installed as described in <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracRepositoryAdmin#ExplicitSync\">TracRepositoryAdmin</a>, and enabling the optional commit updater components by adding the following line to the <tt>[components]</tt> section of your <a class=\"wiki\" href=\"/LUCICodeRepository/nomaticIM/wiki/TracIni#components-section\">trac.ini</a>, or enabling the components in the \"Plugins\" admin panel.\n</p>\n<pre class=\"wiki\">tracopt.ticket.commit_updater.* = enabled\n</pre><p>\nFor more information, see the documentation of the <tt>CommitTicketUpdater</tt> component in the \"Plugins\" admin panel.\n</p>\n<h2 id=\"UsingTrac\">Using Trac</h2>\n<p>\nOnce you have your Trac site up and running, you should be able to create tickets, view the timeline, browse your version control repository if configured, etc.\n</p>\n<p>\nKeep in mind that anonymous (not logged in) users can by default access most but not all of the features.
